# 정상 데이터 룰 발화율 — 우리 vs 실무 벤치마크 비교

작성 2026-07-01. 대상: `datasynth_semantic_v1_normal_20260630_v47_batchid_r7`(정상, truth 0).
목적: 정상 데이터 high 25.9% 가 실무 대비 과한지 판정. 단순 룰 축 + 룰 조합 축 양쪽 비교.

## §1. 단순 룰 비교 (개별 룰 발화율, 전표 단위)

우리 분모 = 111,455 신호전표(≈ 전체). 실무 수치는 근사·범위(출처 §4).

| 룰                          | 우리  | 실무 정상 범위(근사)                                       | 판정                                  |
| --------------------------- | ----- | ---------------------------------------------------------- | ------------------------------------- |
| L3-02 수기입력              | 27.5% | manual JE 라인 ~10%(우수)~30~40%(중하위), APQC             | **정상 범위**(중앙값권)               |
| L1-06 직무분리(SoD)         | 20.2% | SoX "가장 흔한 통제 결함", SME 다발. 정량 공개 적음        | **정상~약간높음**(SME면 타당)         |
| L2-05 역분개                | 36.8% | 발생주의 월별 역분개 routine. 문헌: "드물지만 정상 이벤트" | **정상 상단**(경상적)                 |
| L3-04 기말/기간말           | 45.8% | 결산·기말 집중 정상. 문헌: year-end adjustment 정상        | **정상 상단**                         |
| L1-07 승인생략(승인자 빈칸) | 68.5% | 자동 전표 무승인 정상(실무 ERP 대부분 자동)                | **정상**(단 99% 자동 → 위험신호 아님) |
| L4-04 희소계정쌍            | 6.4%  | 이례 계정조합은 소수                                       | **정상 범위**                         |

**§1 결론: 개별 룰 발화율은 실무 정상 범위 안이다.** 즉 datasynth 는 룰 단위로 현실적이다.
과발화하는 개별 룰은 없다(L1-07 68.5% 도 자동 전표라 위험신호 아님 — batch_id 게이트가 이미 처리).

## §2. 룰 조합 비교 (high 우선순위 비율)

| 지표                  | 우리      | 실무 벤치마크                                           | 판정                     |
| --------------------- | --------- | ------------------------------------------------------- | ------------------------ |
| 정상 데이터 HIGH 비율 | **25.9%** | 실 부정/고위험 전표 <1%(주입 이상 "작은 분율" 0.1~0.5%) | **5~25배 과다**          |
| 상세검사 대상 표본    | (해당)    | 20,000건 → 60건 = **0.3%**                              | 과다                     |
| 중요성 필터 통과      | (해당)    | 20,000건 → 3,000건 = **~15%**                           | 우리 high 가 이보다도 큼 |

**핵심 문헌 근거(Schreyer et al., 회계 이상탐지)**: "전통적 red-flag 테스트는 **역분개(reverse
postings)·충당금(provisions)·기말조정(year-end adjustments)** 같은 **드물지만 정상적인 이벤트**
때문에 **대량 false-positive** 를 내며, 이들은 **부정 위험이 낮다**." → 우리 high 구성(§3)과 정확히
일치. 표준 룰기반 JE 테스트가 "huge number of transactions" 를 flag 하고 다수가 false positive 라는
것도 동일 문헌.

## §3. 왜 조합에서 터지나 (우리 high 구성 대조)

우리 high 7,669 의 주제 구성: 결산 53.2% / 승인통제 22.2% / 자금유출 12.6% / 계정 11.6%.
콤보(중복): 횡령(역분개+승인우회) 80.4% · 결산조작(기말+민감) 65.0% · 희소+승인우회 57.9%.

- high 의 다수가 **역분개·기말** 을 포함 → 문헌이 지목한 "low-fraud-risk false positive" 그 자체.
- 개별로 흔한 신호(수기 27%·SoD 20%·역분개 37%·기말 46%)를 **AND 로 묶으면** 상관성 때문에 union
  이 25% 로 커진다. 실무 고위험 조합은 **서로 독립적이고 드문** 신호의 AND(예: 수기+심야+라운드넘버+
  이례계정 동시)라 <1% 인데, 우리 콤보는 **흔한 신호끼리의 AND** 라 flood.

## §4. 판정

- **단순 룰 축**: 우리 datasynth 는 실무 정상 범위. 개별 룰 과발화 아님.
- **룰 조합 축**: high 25.9% 는 실무 고위험 비율(<1~5%) 대비 **5~25배 과다**. 원인은 개별 룰이 아니라
  **흔한 정상 이벤트(역분개·기말·SME SoD)를 HIGH 콤보 leg 로 묶은 것**. 문헌이 명시한 red-flag
  false-positive 패턴과 동일.
- 함의: (a) HIGH 콤보 정의가 실무 대비 느슨(흔한 이벤트 조합) — HIGH_COMBO_GROUNDING 이 HIGH 로
  정의했으나 실무 false-positive 문헌과 충돌, 또는 (b) 정상 datasynth 가 이 조합 구조를 과생성.
  결정 필요.
