## in/out 잠금규칙 (주입 가능한 조작 전표 실존 여부)
- tier `전표무관` -> **out**.
- `HIGH`/`HIGH(NEW조합)` -> **in** (가공·횡령 전표 실존).
- `LOW` -> **out** (전수확인: 76/76 전원 "전표조작 신호 없음" = 추정/정책/분류판단).
- `MEDIUM` -> 룰조합에 조작전표 rule(L3-02 수기위조·수불조작 / L2-01 한도분할 / L2-02 중복지급·횡령 / L2-03 중복전표 / L2-04 비용자산화 / L2-05 역분개) 있으면 **in**. 비용자산화(자산화/자본화)·순환거래(PHASE1-2 family)도 **in**.
- **수익 과대(commission, weak-signal, 27건)**: 총액/순액 부풀림·cutoff 조기인식은 전표가 실제로 찍혔으므로 **in**. 의도 라벨(가공전표/결산수정) 분리 폐기 — 값이 같으면 같은 부정으로 merge(MEDIUM cutoff∪총액 전수, base_in 제외). 단 정상거래와 전표상 차이가 약해 surface 미검출이 결함 아닐 수 있음 → weak-signal 태그(채점 해석 주의).
- **out = 주입 불가(omission/추정/표시/공시/연결)**: 충당금·손상 *안 잡기*, 총액순액 외 분류판단, 공시 누락, 연결제거. 심을 전표가 없음. 폐기 아님 — **blind-spot 표로 보존**(§10 음의공간).
