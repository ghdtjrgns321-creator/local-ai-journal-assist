# pattern_spec 6 — 순환거래 (circular transaction)

모집단: in-scope 패턴족=순환거래 (multi-label 13). PHASE1-2 family(graph), 상태 중량.
대표 case: `2018-2019-16`(C→E(paper)→C 자금 순환), `2012-가`(해외자회사 출자금 송금→회수→로열티 매출 위장), `2011-가-라`(홍콩 paper company·하청 경유 가공 매출·매입 순환), `2015-가`(3자 순환 허위매출), `A-2유형-나`(3자간 허위매출 순환).

## 기전 (graph / money flow)
A→B→C→A 형태로 자금·매출이 폐쇄 순환한다. 외부 실수요가 없는데 노드 간 거래·세금계산서를 수수해 매출·자산을 부풀린다. 전형: (a) **paper company 경유 자금 순환**(출자금 송금→회수→매출대금 회수로 둔갑), (b) **임원 설립 법인과 고가매입→지분 역취득**, (c) **직원-거래처(employee-vendor) master 연결**. 단일 전표가 아니라 *노드 간 관계*가 부정의 증거 — PHASE1-2 graph surface 담당.

## A — 기전 고정값 (5벌 동일)
| 컬럼                | 값                                            | 근거          |
| ------------------- | --------------------------------------------- | ------------- |
| counterparty_type   | paper company·특수관계자·3자 노드             | 순환 구조     |
| trading_partner     | 순환 노드 식별자(A→B→C→A 폐쇄)                | graph 정체성  |
| gl_account          | 매출·매출채권 / 매입·매입채무 / 출자금·로열티 | 순환 대상     |
| supporting_doc_type | 위조 세금계산서·거래명세표                    | 실물이동 없음 |

## C — 현실 범위 (5벌 재추출)
| 컬럼         | 범위                               |
| ------------ | ---------------------------------- |
| debit_amount | 순환 규모(동일 자금 반복 통과)     |
| posting_date | 상장폐지·관리종목 회피 결산기 집중 |

## D — 상태 대상 (강함 — graph 정합)
- **순환 폐쇄성**: A의 유출이 노드를 거쳐 A로 재유입 — 순환 경로의 전표들이 **동일 자금·동일 폐곡선**으로 연결.
- **노드 간 거래 쌍**: A의 매출 = B의 매입, 금액·시점 대응.
- **employee-vendor**: 직원 master ↔ 거래처 master의 실재 연결(임원이 설립·지배).

## spec 선언위반 (= truth)
1. A→B→C→A 자금·매출이 **폐쇄 순환**(외부 실수요·실물이동 없음).
2. 세금계산서·거래명세표가 **실물이동 없이 수수**된다.
3. 동일 자금이 **출자→회수→매출대금 회수로 둔갑**한다.
4. 임원·직원이 **설립·지배하는 법인과 거래**하면서 특수관계를 은폐한다.
> 개별 전표의 한 줄 정합(차대·계정짝)은 지키되 *관계(graph) 차원*에서만 위반. rule-blind: 위반은 자금 폐곡선·노드 대응·관계 은폐의 사실이다.
