# pattern_spec 3 — 계정분류 misbooking (account misclassification)

모집단: in-scope 패턴족=계정분류 (multi-label 23). Phase 1, 상태 경량~중간.
대표 case: `2014-다`(합병 승계 부실 매출채권을 가공 재고·유형자산으로 대체), `2009-라`(존재하지 않는 원재료 148억 가공계상), `2016-사(재고과대)`(허위 결산조정전표로 가공 재공품·반제품 계상), `2016-아(매입채무과소)`(허위 결산조정전표로 매입채무 과소계상), `2016-가`(순액/총액 오인 수익).

## 기전 (money/document flow)
거래의 경제적 실질과 다른 계정에 기표한다. 두 갈래: (a) **부실·손실 자산을 정상 자산 계정으로 대체**해 손실을 숨김(부실채권→가공 재고/유형자산), (b) **존재하지 않는 자산을 정상 계정으로 가공 계상**(가공 원재료·재공품). 결산조정전표를 수기로 입력하고 수불부·원가명세서를 조작한다. 비용자산화(spec 2)와 인접하나, 여기선 *계정 간 오분류·부실 은폐*가 핵심.

## D — 상태 대상 (부실 대체형은 중간)
(a) 부실자산 대체형: **기존 불량 매출채권·부실 자산 잔액을 참조**해 정상 계정으로 옮긴다(상태 중간 — 실재 부실 객체 필요). (b) 순수 가공계상·총액/순액형: 새 분개라 상태 약함.

## spec 선언위반 (= truth)
1. 거래·잔액이 **경제적 실질과 다른 계정**에 기표된다(부실채권을 재고로).
2. (가공계상형) 계상된 자산에 **대응 실물이 없다**(존재하지 않는 원재료·재공품).
3. 손실·부실을 **정상 자산으로 둔갑**시켜 손실 인식을 회피한다.
4. (총액/순액) 대리인 거래를 본인 기준 **총액 인식**한다(거래 실질과 표시 불일치).
> 그 외 정합은 지킨다. rule-blind: 위반은 계정-실질 대응·자산 실재의 사실이다.
