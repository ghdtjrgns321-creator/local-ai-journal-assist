# 작업: 회계감사 기준 재정독 + 외부 부정사례 검색 → HIGH 주제·조합 발굴

> 한국어로 작업·보고. 읽기 + 웹 검색 + 산출 파일 1개 작성만. 소스 코드/설정/다른 문서 절대 수정 금지.

## 1. 목표
- (a) 회계감사 기준을 다시 정독하고 (b) 외부 회계부정 사례를 권위 출처 중심으로 전수 검색해, **현행 criteria에 아직 없는 HIGH 갈 만한 룰 조합/주제**를 발굴한다. 각 제안을 (기존 5주제 연결 or 신규 주제) + tier + **출처 인용**과 함께 적고, 마지막에 **금감원 사례와 대조**(이 조합이 FSS 패턴에도 나타나나)한다.
- 목적: HIGH 등급 조합의 근거를 감사기준 + 외부 실증으로 넓히고 보강한다.
- 성공 기준: 산출 `dev/active/phase1-rule-basis-audit/p2_standard_external_high_combos.md`에 제안 조합 표(행마다 출처 인용·criteria 중복여부·금감원 대조 포함) + 출처 목록 + 집계가 있고, §6 검증 통과.

## 2. 컨텍스트
- 읽어야 할 기준(in-repo 파싱본): `docs/spec/DETECTION_REFERENCE.md` §2 전체(감사기준서 240 §32·A44·A45·A46 / 315 §26 / 520 §5 / 550 §23 / 330 / 500 / 1100, 외감법 §8·K-SOX).
- 비교 대상(이미 있는 criteria — 중복 발굴 금지용, 반드시 먼저 읽어라): `docs/spec/DETECTION_RANKING_CRITERIA.md` §점수 근거 요약·§High 승격 가능 조건·§High 금지 또는 약화 조건, `docs/spec/PHASE1_TIER_EVIDENCE_BASIS.md` §4~§5.
- 금감원 대조용: `docs/spec/DETECTION_REFERENCE.md` §3.3(6대 패턴) + (있으면) `dev/active/phase1-rule-basis-audit/fss_case_combo_tagging.md`(프롬프트1 산출).
- 외부 검색 출처(웹, 권위 중심 — 전수로 훑어라): SEC AAER(Accounting and Auditing Enforcement Releases) 부정 유형, ACFE Report to the Nations 점유 사기 scheme 분류, COSO Fraudulent Financial Reporting 연구, PCAOB AS 2401 / ISA 240 부록 예시, 학술 journal-entry fraud taxonomy. (블로그·홍보성 자료 제외, 1차/권위 출처만.)
- **현행 위험 주제 5개**: 승인통제 / 결산시점 / 계정분류 / 중복자금유출 / 수익통계. (원장정합성=데이터 트랙 분리, 순환거래=PHASE1-2 family.)
- **현행 HIGH 조합(이미 있음 — 재탕 금지, 비교만):**
  - 가공전표→수익통계: `(L4-01 or L4-03)+L3-02+(L4-04 or L2-03)`
  - 결산수정→결산시점: `(L3-04 or L3-07 or L3-11 or L1-08)+L4-03+(L3-08 or L3-10 or L4-04)`
  - 횡령은폐→중복자금유출: `(L2-02 or L2-03 or L2-05)+(L1-04 or L1-05 or L1-06 or L1-07)`
  - 승인우회→승인통제: `(L1-04 or L1-05 or L1-06 or L1-07)+강한보강(L4-03 or L3-11 or L3-04+L3-02 or L3-06+L3-02)`
- **룰 카탈로그(이 id만. 발명 금지):** L1-04 승인한도·L1-05 자기승인·L1-06 SoD·L1-07 승인생략·L1-08 기간불일치·L1-09 승인일누락·L2-01 한도직하·L2-02 중복지급·L2-03 중복전표·L2-04 비용자산화·L2-05 역분개·L3-01 계정분류불일치·L3-02 수기전표·L3-03 관계사맥락·L3-04 기말결산·L3-05 주말공휴일·L3-06 심야·L3-07 전기일괴리·L3-08 적요결손·L3-09 가수금장기체류·L3-10 고위험계정·L3-11 매출컷오프·L3-12 업무범위집중·L4-01 매출이상변동·L4-03 이상고액·L4-04 희소계정쌍·L4-05 비정상시간·L4-06 배치. (L1-01/02/03=데이터 트랙, 조합 재료 금지.) *주제·조합은 신규 생성 허용, 룰 id만 고정.*

## 3. 설계 (이대로 — 임의 변경 금지)
1. §2의 기준서 파싱본을 정독하고, 기준서가 "elevated/significant risk"로 다루는 **특징 조합**(예: 비경상계정+기말+설명부족)을 추려 룰 id 조합으로 변환.
2. 외부 출처(SEC AAER/ACFE/COSO/PCAOB·ISA 예시)를 전수 훑어, 반복되는 부정 **수법 패턴**을 룰 조합으로 변환.
3. 각 후보를 현행 criteria(기존 4 HIGH 조합·금지조합)와 대조: **이미 있으면 제외**(중복), 변형/강화면 "기존 보강"으로 표시, 완전 새로우면 "신규".
4. 각 신규/보강 조합을 기존 5주제에 연결. 어디에도 안 맞으면 `NEW:<주제명>` 신규 주제 생성.
5. tier 판정: 기준서·외부사례가 "강한 복수 증거"로 다루면 HIGH, 단일·약한 정황이면 MEDIUM 후보. (PCAOB AS 2401 §61 "복수 특성" 원칙 적용.)
6. **마지막 — 금감원 대조**: 각 제안 조합이 §3.3 6대 패턴(또는 프롬프트1 산출)에 실제로 나타나는지 표시(`FSS:가공전표` 등 / `FSS 없음`). FSS에도 있으면 근거가 더 강함.
- 룰로 표현 불가한 구조 패턴(순환거래류)은 "PHASE1-2 family 후보"로 분류하고 PHASE1-1 조합으로 억지 변환하지 마라.

## 4. 단계 체크리스트 (순서 고정 — 건너뛰기·합치기 금지)
- [ ] Step 1: §2 기준서 파싱본 + §비교대상 criteria 정독. 증거: 기존 4 HIGH 조합을 그대로 적어 "읽었음" 확인 + 기준서에서 추린 특징조합 후보 메모.
- [ ] Step 2: 기준서 기반 조합 후보 도출 → 표에 기입(출처=기준서 조문). 증거: 표 발췌 + 각 행 조문 인용.
- [ ] Step 3: 외부사례 웹 검색(SEC AAER/ACFE/COSO/PCAOB·ISA) 전수 → 수법 패턴 → 조합 후보 표 기입(출처=URL/문헌). 증거: 검색한 출처 목록 + 표 발췌.
- [ ] Step 4: 현행 criteria 대조(중복/보강/신규 분류) + 주제 연결(기존5 or NEW). 증거: 분류 결과.
- [ ] Step 5: 금감원 §3.3(+프롬프트1 산출) 대조 칸 채움. 증거: 대조 결과 표.
- [ ] Step 6(마지막): 집계 — 신규 HIGH 조합 N개, 신규 주제 N개, 기존 보강 N개, FSS 동시확인 N개. + §6 검증.
※ 출처 인용 없는 제안 행 = 미수행.

## 5. 금지 사항 (1건이라도 위반 시 작업 전체 실패)
- **출처 없는 제안 금지**: 모든 행에 기준서 조문 **또는** 외부 출처(URL/문헌) 인용. 둘 다 없으면 그 행 삭제.
- **criteria 재탕 금지**: §2의 기존 4 HIGH 조합을 "신규"로 다시 제안 금지(중복 칸에 명시).
- **룰 id 발명 금지**(주제·조합은 신규 허용).
- **fitting 금지**: DataSynth truth·라벨·성능에 맞춰 조합 제안 금지. 근거는 기준서·외부사례·금감원만.
- **추측 출처 금지**: 실제로 읽은/검색한 출처만 인용. URL 지어내기 금지.
- 소스/설정/타 문서 수정 금지(산출 1파일). 체크리스트 순서변경·생략 금지. 미완을 완료로 보고 금지.
