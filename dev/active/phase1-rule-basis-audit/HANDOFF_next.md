## 숲 — 지금 뭘 하고 있나

PHASE1 위험도 등급(HIGH/MEDIUM/LOW/CONTEXT)이 "왜 그 등급인지" **근거(금감원 실증 + 감사기준)**를 만드는 작업.

진행 경로:
1. HIGH 조합을 5개로 확정 + 근거 문서(`high_combo_grounding.md`) 깊이 재작성 → 완료
2. 사용자 지적: "HIGH 5개는 너무 적다. 금감원 자료 다 봤을 때 저것밖에 안 나와?"
3. 원천 전수 재집계 → **HIGH 158건 중 17건이 기존 5조합 밖(`HIGH(NEW조합)`)으로 따로 태깅돼 있었음.** 내 문서가 빠뜨린 누락(정직 자백 완료).
4. 17건 원문(무슨 수법인가)을 까서 "새 조합이냐 / 셋째 다리만 다르냐" 분석 → **거의 전부 같은 스토리, 셋째 다리만 다름**.

## 구조적 발견 2가지 (핵심)

**① 조합2가 "승인룰 필수"라 너무 좁다 (★)**
조합2 = 자금유출룰 + 승인룰(L1-04~07). 그런데 감리2013-1-가·A-6유형-가는 명백한 은폐 스토리인데 승인 신호가 안 떠서 탈락(역분개 L2-05 + 수기 L3-02 + 고액으로만 나타남). 횡령은폐가 항상 승인 흔적을 남기는 게 아니다.
→ 조합2 셋째 다리를 **승인룰 OR (역분개/중복 + 수기)**로 완화해야 함.

**② 결국 메타패턴은 하나다 — "수기 + 고액 + 2차 정황 1개"**
조합1·2를 셋째 다리까지 풀면 사실상:
```
HIGH = L3-02(수기) + 고액(L4-03 또는 매출급변 L4-01)
       + [2차 비정상 1개: 희소계정·중복·관계사·고위험계정·기말·통제이상·컷오프 중 하나]
```
이건 **PCAOB AS 2401 §61**("복수 위험특성이 겹치면 고위험")의 정확한 구현. 지금은 그 "2차 정황" 풀을 좁게 손으로 나열해서 11건을 흘린 것.

## ▶ 결정 필요 (§8 게이트 — 다음 세션 첫 단계)

12건(+오태깅 2건)을 어떻게 처리할지 = 사용자 결정.

- **A안 (추천) — 셋째 다리 확장**: 조합1·2의 OR 분기에 관계사(L3-03)·기말(L3-04)·고위험계정(L3-10)·자기승인(L1-05)·승인일자공백(L1-09)·컷오프(L3-11) 추가. 신규 조합 안 만들고 **기존 조합의 인정 정황을 FSS 실증에 맞게 확장**. → 11건 대부분 흡수. + 조합2에 (역분개 L2-05 + 수기 L3-02) 분기 추가.
- **B안 — 메타규칙 일반화**: "수기(L3-02) + 고액(L4-03) + 아무 2차정황 1개 = HIGH". 실증에 가장 충실하나 약한 정황까지 HIGH로 올라 **과탐 위험 큼**.
- **C안 — 신규 조합 2~3개 추가**: A·B·C 갈래를 각각 독립 HIGH 조합으로. 명시적이나 조합 수 증가·관리 부담.

> ※ 분석 결론(스토리 동일)은 A안을 가리킴. 단 최종 채택은 사용자 결정 — 착수 전 §8 게이트.
