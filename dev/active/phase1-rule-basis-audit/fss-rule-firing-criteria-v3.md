# FSS 사례별 룰 발화 지도 기준서 v3 (tier 폐기)

> 목적 변경: v2까지는 각 FSS 사례에 tier(HIGH/MEDIUM/LOW)를 매겼으나, tier는
> (a) 원문 고의 단어 매칭이라 애매하고 (b) 우리 시스템의 검토 우선순위 tier와 다른 축이며
> (c) 조합이 tier를 정하는데 그 tier로 조합을 검증하는 순환을 만든다. **tier를 폐기한다.**
>
> v3의 유일한 질문: **"이 FSS 사례가 우리 GL에 들어왔다면 어떤 결정론 룰이 발화하는가."**
> 확실한 건 확정으로 명시(코드/원문 근거), 애매한 건 사람이 판정하도록 후보로 올린다.
> 이 발화 지도 위에서 이후 "어떤 룰 조합을 HIGH 큐로 올릴까"를 별도로 설계한다.

## 1. 산출 스키마 (열 고정)

```
| case_id | source_file | 위반 핵심사실(원문 발췌) | 확정룰 | 판단필요룰 | 근거 |
```

- **tier 컬럼 없음. 고의성·심각도 판정 없음.** 심각도는 이 문서의 관심사가 아니다.
- **확정룰**: 원문 발췌 문언에 그 룰의 정의 신호가 직접 있어 발화가 명백한 룰.
  형식 `L3-04(E:"기말 결산 시")` — 따옴표 안은 원문에서 **한 글자도 바꾸지 않고 복사**(기계 대조 대상).
  복수는 공백으로 나열. 없으면 공란.
- **판단필요룰**: 정황상 발화 후보이나 원문 문언이 없거나 룰 정의 경계라 사람 판정이 필요한 룰.
  형식 `L4-01(?:매출 과대계상은 명시되나 규모(금액·비율) 서술 없음)` — 괄호에 **왜 애매한지** 1줄.
  복수는 공백으로 나열. 없으면 공란.
- **근거**: 확정/판단필요 판정을 요약한 한 줄.

판정 규칙:
- 발화가 **명백**(원문 문언 직접) → 확정룰.
- 발화 **가능성은 있으나 문언 부재·경계** → 판단필요룰(사람이 O/X).
- 발화 안 함 → 아무 칸에도 안 씀.
- 24룰 어디에도 안 걸림 → 확정·판단필요 모두 공란 + 근거에 `대응 룰 없음: <경계 유형>`
  (경계 유형은 §4 참조 — 실물수량/실재성/부외/증빙위조/타사원장/공시/회계판단/GL축부재).

## 2. 태깅 대상 룰 (24) / 금지 id (9)

- 태깅 대상: L1-04, L1-05, L1-06, L1-07, L1-07-02, L1-08, L2-01, L2-02, L2-03, L2-04,
  L2-05, L3-02, L3-03, L3-04, L3-05, L3-06, L3-07, L3-09, L3-10, L3-11, L3-12, L4-01, L4-03, L4-04
- 금지: L3-01·L3-08·L1-09(폐기), L4-02·L4-05·L4-06(PHASE1-2 이관), L1-01·L1-02·L1-03(정합성 트랙)

## 3. 룰별 발화 신호 (확정 기준)

각 룰이 "확정"되려면 원문 발췌에 아래 신호 문언이 실제로 있어야 한다. 없으면 판단필요 또는 미태깅.

### L1 승인·통제 (원문이 전표 메타데이터를 거의 안 써서 확정 드묾)

| 룰       | 확정 신호 (원문 문언)                        | 판단필요 사례                                  |
| -------- | -------------------------------------------- | ---------------------------------------------- |
| L1-04    | 전결규정 위반·권한 없는 자의 승인 명시       | "대표이사 지시"만 있고 승인 절차 서술 없음     |
| L1-05    | 기안자=승인자 동일·"스스로 결재/승인" 명시   | "대표이사가 직접 처리"만 있고 승인 무력화 불명 |
| L1-06    | 상충 업무 겸임(구매+지급)·통제조직 폐지 명시 | 1인 지배에서 겸직 정황만                       |
| L1-07    | "승인 없이"·"결재 없이" 명시                 | 승인 언급 부재를 생략으로 추정                 |
| L1-07-02 | 허위 명의 결재·가공 승인자 기재 명시         | 문서 위조 일반                                 |
| L1-08    | 거래 발생 기간과 다른 회계기간 계상 명시     | 수익 인식시점 위반(→L3-11 경계)                |

### L2 금액·패턴 (원문 사실관계로 판정 가능)

| 룰    | 확정 신호                                       | 판단필요 사례                   |
| ----- | ----------------------------------------------- | ------------------------------- |
| L2-01 | 한도 회피 목적 금액 분할 명시                   | 단순 다건                       |
| L2-02 | 동일 대금 이중 지급 명시                        | 반복 지급·우회 이동             |
| L2-03 | 동일 거래 이중 계상(재기표) 명시                | 반복 허위 계상                  |
| L2-04 | 비용성 지출을 자산(개발비·무형자산 등)으로 계상 | 손상·평가 등 자산 과대계상 일반 |
| L2-05 | 반대분개·취소·환입·차기 반제 명시               | 기말 분식 후 차기 반대처리 구조 |

## 4. 대응 룰 없음 — 경계 유형 (근거 칸에 명시)

24룰에 안 걸리는 사례는 아래 경계 유형 중 하나를 근거에 적는다(fsscombo.md §1 A분석과 연결).

실물수량 · 실재성(외부조회) · 부외거래 · 증빙위조 · 타사원장 · 공시/주석 · 회계판단(평가·분류·정책) · GL축부재(프로젝트/거래처)

## 5. 절차·검증 (태깅과 검증 분리)

1. 태깅: 배치 에이전트(sonnet)가 230파일 전수. 행마다 발췌 필수.
2. 기계 검증: (a) 230파일 커버리지 0 누락 (b) 금지 룰 id 0건 (c) 확정(E) 인용 원문 대조 —
   txt=원문 전문, PDF=pypdf 추출 전문, 문장부호 정규화. 불일치 전건 목록 (d) 판단필요룰 전건 사유 존재.
3. 재현성 표본: 무작위 50건 독립 에이전트 재태깅 → 확정룰 집합 일치율 보고.

## 6. 이 지도의 사용처 (다음 단계, 이 문서 범위 아님)

- 룰 발화 지도가 완성되면 → "확정룰이 함께 뜨는 조합"을 사례 간 집계 → 반복 조합 도출
- 각 조합의 정상 데이터 과탐율 측정 → HIGH 큐 승격 조합 결정
- **tier는 조합 설계 결과로만 나온다(사례에 미리 안 매긴다).**
