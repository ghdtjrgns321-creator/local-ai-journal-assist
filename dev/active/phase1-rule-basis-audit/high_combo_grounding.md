## 이 문서가 답하는 질문

1. HIGH / MEDIUM / LOW / CONTEXT가 각각 무엇인가, 무엇으로 갈리나 → **§1**
2. tier(등급)를 아예 안 받는 룰도 있나 → **§2**
3. 등급 근거를 어디서 가져왔나 → **§3, §4**
4. 지금 "HIGH"로 확정된 전표 조합 5개는 각각 무슨 패턴이고 왜 그게 근거가 되나 → **§5 (핵심)**
5. 기준서가 제안했는데 안 받은 조합은 왜 버렸나 → **§6**

## §0. 먼저 — PHASE1이 tier로 하는 일

PHASE1은 부정(fraud)을 **확정하는 단계가 아니다.** 수만 건의 전표 중에서 "감사인이 먼저 봐야 할 것"을 위로 끌어올려 **우선순위 줄을 세우는 단계**다. 그 줄 세우기의 이름표가 tier다.

- **HIGH** = 가장 먼저 보라 (여러 위험 정황이 한 전표에 겹침)
- **MEDIUM** = 그 다음 보라 (위험 정황이 하나는 확실함)
- **LOW** = 시간 되면 보라 (신호 하나가 떴으나 겹치는 정황 없음)
- **CONTEXT** = 단독으로는 줄에 못 섬 (다른 신호를 거들어주는 맥락일 뿐)

### 가장 흔한 오해 한 가지

> tier는 **점수가 높아서** HIGH가 되는 게 아니다.

tier는 **"어떤 위험 트리거가 떴는가 + 그 주제를 단독으로 세울 primary 룰이 있는가"** 로 갈린다 (`compute_topic_tiers`, topic_scoring.py:369).

코드에 floor 값 0.45·0.75가 보이지만, 이건 가중합 점수의 커트라인이 아니다. **"이 트리거가 떴으면 MEDIUM 칸, 저 트리거가 떴으면 HIGH 칸"으로 보내는 순서표일 뿐**이다 (`_floor_value_tier`: 값 ≥0.75면 HIGH, ≥0.45면 MEDIUM). 즉 0.75는 "75점"이 아니라 "HIGH 칸"이라는 라벨에 가깝다.

## §1. tier 4종 — 각각 무엇이고 어디서 근거를 채택했나

표 한 줄로 압축하면 읽히지 않아, 등급마다 풀어 쓴다.

### HIGH — "여러 정황이 한 전표에 겹쳤다"

- **언제 붙나**: §5의 HIGH 조합 트리거(floor ≥0.75)가 발화하고, 그 주제를 단독으로 세울 primary 룰이 같이 있을 때.
- **무슨 의미**: 위험 신호 하나가 아니라 **서로 다른 종류의 정황 2개 이상이 같은 전표에 모인** 사기 패턴. 예: "이상하게 큰 금액(L4-03)" + "수기로 입력(L3-02)" + "매출이 급변(L4-01)"이 한 전표에 동시에.
- **근거 출처**: 이 문서 §4·§5. **한국 FSS 실제 사례(P1)** 로 "이런 조합이 실제 분식·횡령에서 나왔다"를 확인하고, **감사기준서(P2)** 로 "기준서도 이 조합을 고위험으로 본다"를 교차 확인. 두 축이 다 충족된 5개 조합만 HIGH로 고정.
- **기준서 핵심 한 줄**: PCAOB AS 2401 §61 — "부정 위험은 **복수의 위험특성이 겹칠 때** 비로소 높다."

### MEDIUM — "확실한 정황이 하나는 있다"

- **언제 붙나**: MEDIUM 조합 트리거(floor ≥0.45)가 발화하고 primary 룰이 있을 때. 강한 정황 1개, 또는 약한 조합.
- **무슨 의미**: 봐야 할 이유는 분명하지만 HIGH만큼 겹치진 않은 전표.
- **근거 출처**: PCAOB AS 2401 §61("복수=HIGH, 단일=주의") + 정상 데이터에서도 단일 신호는 흔히 나온다는 관찰(DETECTION_RANKING_CRITERIA). 조합의 구체적 분기 기준은 §5 각 조합의 "medium 분기" 설명.

### LOW — "신호 하나만 단독으로 떴다"

- **언제 붙나**: 조합 없이 **primary 룰 1개만 단독** 발화.
- **무슨 의미**: 룰 하나가 떴지만 그것을 거들어주는 다른 정황이 없음. 정상 업무에서도 흔히 나오는 단일 신호일 수 있음.
- **근거 출처**: **그 룰 자체의 발화조건.** 대부분 ISA 240 A45나 외감법 §8에 닿는다(=[STD], phase1-rule-basis-audit.md §3). 정리하면 "근거는 룰의 발화조건에 있고, 단독이라서 등급만 LOW".

### CONTEXT — "혼자서는 줄에 못 선다"

- **언제 붙나**: booster(보강)·macro(거시)·combo_only 신호만 있고 **primary 룰이 없을 때.**
- **무슨 의미**: 다른 신호가 떴을 때 그 위험도를 거들어주는 맥락 정보. 단독으로는 검토 큐에 못 올라간다.
- **예**: 관계사 거래 맥락(L3-03), 거시 통계 신호(D01·D02·Benford·L4-02). 이들은 PHASE1-2(family) 또는 보강용으로 이관됨.

## §2. tier를 아예 안 받는 룰 — 3종류

모든 룰이 위험 등급을 받는 게 아니다. 세 부류는 HIGH/MEDIUM/LOW 줄 밖에 있다.

### (1) 데이터 정합성 트랙 — L1-01·L1-02·L1-03

- **무슨 룰**: 차대불일치(L1-01), 필수필드 누락(L1-02), 계정과목표(CoA) 밖 계정(L1-03).
- **왜 등급이 없나**: 2026-06-15 사용자 결정으로 **위험 큐에서 통째로 분리.** 이건 "부정 위험"이 아니라 **"데이터가 분석 가능한 상태인가"** 의 문제이기 때문. 차변·대변이 안 맞거나 금액 칸이 비었으면, 그건 사기 신호가 아니라 데이터 품질 결함이다.
- **어떻게 표시되나**: tier 없이 별도 "데이터 품질" 집계(`metadata.data_integrity_findings`)로 건수만 보여줌.
- **근거**: phase1-rule-basis-audit.md §5-1.

### (2) 발화조건조차 기준서 근거가 없는 룰 — L1-09·L4-05·L3-10

이 셋은 tier 안에서 돌긴 하지만, **"애초에 왜 이 신호를 보는가"의 기준서 근거가 가장 빈약**하다. (Task 2의 제거 검토 후보)

```
L1-09  승인일자 공백   → L1 9개 룰 중 유일하게 인용 조문이 없음.
                         "추적성이 훼손된다"는 일반론만 있음.        [근거 약함]
L4-05  비정상시간 집중 → KLCA IT 체크리스트에 해당 조문·항목 없음.
                         31개 룰 중 발화근거가 가장 빈약.            [근거 약함]
L3-10  고위험계정 사용 → 규정 인용 없이 "감사인이 지정한다"로 위임.   [관행]
```

### (3) 임계 숫자·점수 계수 차원에서는 거의 전 룰이 [근거없음]

발화조건("무엇을 보나")은 대부분 기준서에 닿는다. 그러나 **"얼마나·어떤 코드냐"** — z>3, 10억 원, ±5일, aging 30일, 차단계정 45개, severity 1~5 같은 **구체 숫자는 거의 전부 근거가 없거나 도메인 디폴트**다. 이 결손 목록이 phase1-rule-basis-audit.md §2의 작업 백로그(1~4순위)이고, Task 4의 대상이다.

> 정리: **무엇을 보는가는 근거 있음, 얼마나/어떤 숫자로 보는가는 근거 없음** — 이게 전 룰 공통 패턴이다.

## §3. 근거를 어디서 가져왔나 — 두 출처

| 출처 | 무엇인가 | 무엇을 보장하나 |
|------|----------|----------------|
| **P1** (FSS 실증) | 금감원 감리지적 230파일 → 474 사례 태깅 | "한국에서 **실제로** 이 조합이 분식·횡령에서 나왔다" |
| **P2** (기준서·외부) | ISA 240·PCAOB·외감법 재정독 + 해외사례 | "**기준서·이론도** 이 조합을 고위험으로 본다" |

P1의 474 사례 등급 분포:

```
HIGH      158건
MEDIUM    154건
LOW        78건
N/A        84건   ← 전표와 무관(공시·주석 영역 등)
```

**왜 두 출처를 다 봐야 하나**: P1만 보면 "한국 특수 사례에 끼워 맞춘 것"이 될 수 있고, P2만 보면 "교과서적이지만 한국 현실엔 없는 패턴"을 넣을 수 있다. **둘 다 충족한 조합만 HIGH로 고정**하면 양쪽 위험을 모두 막는다. 실제로 §6에서 보듯 P2가 "고위험"이라 한 조합 여럿이 P1 실증에서 0건이라 탈락했다.

## §4. P1 실증 골격 — HIGH 사례에서 실제로 무엇이 같이 떴나

조합을 머리로 가정하지 않고, **실제 HIGH 158건에서 어떤 룰이 함께 나타났는지 빈도부터 셌다.**

**단일 룰 빈도 (HIGH 158건 중 등장 횟수)**

```
L3-02 수기전표        116      L3-04 기말결산전표      44
L4-03 이상고액        108      L2-03 중복전표          30
L4-01 매출이상변동     60      L4-04 희소계정쌍        27
L2-02 중복지급         49      L3-11 컷오프/반복        22
L1-05 자기승인         45      L3-03 관계사맥락         18
```

**룰 3개가 동시에 뜬 조합 TOP (실제 동시 출현 횟수)**

```
L3-02 + L4-01 + L4-03  = 30 건    ← 가공전표 골격
L2-02 + L3-02 + L4-03  = 24 건    ← 횡령은폐 골격
L3-02 + L3-11 + L4-01  = 19 건
L1-05 + L2-02 + L4-03  = 19 건    ← 자기승인이 횡령 조합에 결합
L1-05 + L3-02 + L4-03  = 19 건
```

**읽는 법**: HIGH의 실증 골격은 크게 두 줄기다.

1. **가공전표 줄기** — 수기입력(L3-02) × 이상고액(L4-03) × 매출급변(L4-01). 없는 매출을 사람이 손으로 크게 만들어 넣은 흔적.
2. **횡령은폐 줄기** — 중복지급(L2-02) × 수기입력(L3-02) × 이상고액(L4-03). 돈이 두 번 빠져나간 흔적.

그리고 자기승인(L1-05)은 **독립된 줄기가 아니라** 위 두 줄기에 얹혀 자주 같이 나온다. 이게 §5에서 "승인우회를 독립 주제에서 빼고 보강으로 흡수"한 실증 근거다.

## §5. 확정된 HIGH 조합 5개 — 조합별 깊은 해설 (핵심)

각 조합마다: **무슨 패턴인지(일상어) → 룰 조합(코드블록) → FSS 실증 → 기준서 근거 → 도메인 의미 → 왜 HIGH인가** 순서로 푼다.

룰 약어:

```
L1-04 승인한도 초과   L2-02 중복지급      L3-09 가수금 장기체류
L1-05 자기승인        L2-03 중복전표      L3-10 고위험계정 사용
L1-06 직무분리 위반   L2-04 비용→자산화   L3-11 컷오프 위반
L1-07 승인 생략/공백  L2-05 역분개        L4-01 매출 이상변동
L3-02 수기전표        L3-04 기말결산전표   L4-03 이상고액 전표
L3-08 적요 부실       L4-04 희소계정쌍
```

### 조합 1 — 가공전표 → 수익통계 [강근거, 유지]

**무슨 패턴인가**: 없는 매출을 사람이 손으로 크게 만들어 넣은 전표. "매출을 부풀리는" 분식의 가장 전형적 형태.

```
조합:  (L4-01 또는 L4-03)  +   L3-02   +  2차정황 1개
        매출급변   이상고액      수기입력
        ── 둘 중 하나 ──        (사람손)
  2차정황 = L4-04 희소계정쌍 / L2-03 중복전표 / L3-03 관계사 /
            L3-10 민감계정 / L1-05 자기승인 / L3-11 cutoff   중 하나
            (A안 확장 §5b — 코드 반영 완료)
```

> **A안 셋째 다리 확장 (2026-06-16 설계 → 2026-06-17 코드 반영)**: 2차정황 풀이 원래 `(L4-04 또는 L2-03)` 2종뿐이라, 같은 가공전표 스토리인데 관계사 경유·자기승인 등 다른 다리로 나타난 FSS 사례가 그물을 빠져나갔다(§5b 17건 감사). 신규 조합을 만들지 않고 인정 2차정황을 넓혔다. **단 후보였던 L3-04(기말)·L1-09(승인일공백)는 과탐 측정에서 정상 결산전표에 과발화(기말 734/738)해 제외**(§5b 과탐 가드). 최종 코드 2차정황 = `L4-04·L2-03·L3-03·L3-10·L1-05·L3-11` (`topic_scoring.py::_FICTITIOUS_SECONDARY_RULES`).

**FSS 실증**: 474건 중 **46건 매칭, 그중 44건이 HIGH.** P1에서 가장 두꺼운 골격(§4의 1줄기).

**기준서 근거**:
- ISA 240 A45(a) — 비경상적·수동 분개는 부정의 주요 통로.
- PCAOB AS 2401 §61 — 복수 위험특성 동시 발생 시 고위험.

**도메인 의미**: 정상 매출은 시스템이 자동으로(주문→출하→청구) 만든다. 그런데 **사람이 수기로(L3-02), 이상하게 큰 금액을(L4-03), 매출이 급변한 시점에(L4-01), 평소 안 쓰는 계정짝으로(L4-04)** 넣었다면 — 이건 정상 거래의 모습이 아니라 매출을 만들어낸 모습이다.

**왜 HIGH인가**: 위 4가지가 한 전표에 겹칠 정상적 이유를 찾기 어렵다. 한국 실증 46건 + 기준서 2축이 모두 받친다.

### 조합 2 — 횡령은폐 → 중복자금유출 [강근거, 유지]

**무슨 패턴인가**: 같은 돈을 두 번 내보내거나, 내보낸 걸 되돌리는 척하며 빼돌린 흔적. + 그걸 본인이 승인.

```
조합:  (L2-02 또는 L2-03 또는 L2-05)  +  통제이상
        중복지급  중복전표  역분개
        ──── 자금이 새는 통로 ────
  통제이상 = 승인우회(L1-04 한도/L1-05 자기승인/L1-06 직무분리/L1-07 승인생략 중 하나)
           또는 (L2-05 역분개 + L3-02 수기)   ← A안: 승인흔적 없는 은폐 분기 (§5b)
```


**왜 0건인가 — 두 가지 다른 이유**

1. **한국에 그 구조가 없음**: topside(L3-12, 연결 통합조정 전표)는 어떤 FSS 사례에도 안 나온다. 한국 감리 대상이 대부분 단일법인이라 연결단 통합조정 분식이 드물기 때문. → 한국 실증 없음이 맞다.

2. **룰 조합 정의가 실제 사례와 어긋남**: 가공거래처(B-5)는 FSS에 수법으로는 흔하다. 그런데 우리가 잡으려 한 조합 `L3-02+L4-04+L3-08`로는 0건이다. 거래처 위조가 "희소계정쌍(L4-04)·적요부실(L3-08)" 형태로 안 잡히기 때문. → **수법은 있는데 우리 룰 조합이 그 수법을 못 잡는다.** 룰 조합 재설계가 필요한 별도 과제(Task 4 후보).

**처분**: 이 9개는 HIGH로 올리지 않고 **LOW로 약하게만 채택**(floor 미부여 = 코드상 신규 조합 추가 없음). 근거표에 "한국 미실증"으로 정직하게 명시. 가공거래처 룰조합 재설계는 별도 플래그.
