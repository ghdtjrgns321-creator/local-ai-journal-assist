# L3-11 DataSynth 2022~2024 Label Gap Analysis

대상 데이터: `data/journal/primary/datasynth` v45 production baseline

분석일: 2026-04-26

## 기준

- 탐지 룰: `src/detection/evidence_rules.py::ev02_cutoff_violation()`
- 매출 계정: `gl_account` prefix `4`, 허용 영업일 `5`
- 비용 계정: `gl_account` prefix `5`, 허용 영업일 `7`
- 기준일: `posting_date` vs `delivery_date`
- 비교 단위: `document_id`
- 분석 범위: `journal_entries_2022.csv`, `journal_entries_2023.csv`, `journal_entries_2024.csv`

## 요약

현재 DataSynth v45에는 L3-11 전용 정답 라벨(`RevenueCutoffMismatch`, `revenue_cutoff_error` 등)이 없다. 또한 L3-11이 사용하는 `delivery_date`도 대부분 비어 있어 실제 탐지 후보가 0건이다.

| year | rows | docs | valid delivery docs | flagged docs | cutoff truth docs | TP | FP | FN |
|---:|---:|---:|---:|---:|---:|---:|---:|---:|
| 2022 | 373,425 | 106,675 | 85 | 0 | 0 | 0 | 0 | 0 |
| 2023 | 366,465 | 105,525 | 0 | 0 | 0 | 0 | 0 | 0 |
| 2024 | 369,545 | 106,993 | 0 | 0 | 0 | 0 | 0 | 0 |
| ALL | 1,109,435 | 319,193 | 85 | 0 | 0 | 0 | 0 | 0 |

## Evidence Column Coverage

| year | valid `delivery_date` rows | valid `delivery_date` docs | revenue valid rows | expense valid rows | max business-day diff |
|---:|---:|---:|---:|---:|---:|
| 2022 | 170 | 85 | 0 | 24 | 0 |
| 2023 | 0 | 0 | 0 | 0 | 0 |
| 2024 | 0 | 0 | 0 | 0 | 0 |

2022의 유효 `delivery_date` 85문서도 모두 `posting_date`와 같은 날이다. 비용 계정 유효 row 24건의 영업일 차이도 모두 0이라 L3-11 기준을 넘지 않는다.

## Label Coverage

`anomaly_labels.csv`에서 cutoff 관련 라벨은 발견되지 않았다.

| search basis | count |
|---|---:|
| `RevenueCutoffMismatch` anomaly_type | 0 |
| `revenue_cutoff_error` / `expense_cutoff_error` metadata | 0 |
| cutoff 키워드가 포함된 anomaly label | 0 |

참고로 비슷해 보이는 라벨은 존재하지만 L3-11의 정답으로 쓰기 어렵다.

| anomaly_type | 2022 | 2023 | 2024 | total |
|---|---:|---:|---:|---:|
| RevenueManipulation | 2 | 5 | 10 | 17 |
| WrongPeriod | 24 | 15 | 32 | 71 |
| RushedPeriodEnd | 12 | 10 | 14 | 36 |
| LatePosting | 20 | 23 | 73 | 116 |

이 라벨들은 각각 고액/기간/기말/전기 지연 성격이고, `posting_date`와 수익 인식 근거 이벤트일(`delivery_date`)의 cutoff 불일치 정답은 아니다.

## 과탐/미탐 판단

- 과탐: 0건. L3-11 후보 자체가 0건이다.
- 미탐: cutoff truth 라벨이 없어서 정답 기준 미탐은 계산할 수 없다.
- 실질 결론: 현재 v45 합성데이터는 L3-11 검증 데이터셋이 아니다. 탐지기가 실패했다기보다, 정답 라벨과 근거 이벤트일이 주입되지 않았다.

## 원인

1. `anomaly_labels.csv`에는 L3-11에 대응되는 `RevenueCutoffMismatch` 라벨이 없다.
2. `journal_entries_2023.csv`, `journal_entries_2024.csv`는 `delivery_date`가 전부 null이다.
3. `journal_entries_2022.csv`도 유효 `delivery_date`가 85문서뿐이고, 매출 계정 유효 row는 0건이다.
4. document flow JSON에는 `requested_delivery_date` 같은 계획일은 일부 있지만, L3-11 룰이 참조하는 `delivery_date`는 대부분 null이다.

## 권장 조치

DataSynth에 L3-11 전용 truth를 추가해야 한다.

- `RevenueCutoffMismatch` 또는 동등한 anomaly label을 추가한다.
- 매출 전표의 `delivery_date`를 채우고, 일부는 `posting_date`와 5영업일 초과 차이가 나도록 주입한다.
- 비용 cutoff도 검증하려면 비용 계정에 대해 7영업일 초과 케이스를 별도 주입한다.
- 정상 negative control도 함께 만든다: 0~5영업일 매출, 0~7영업일 비용, `delivery_date` 결측으로 검증 불가인 케이스.
- metadata에 `intended_delivery_date`, `applied_posting_date`, `actual_business_day_diff`, `cutoff_truth_type`을 남겨 후속 품질 게이트에서 라벨/실제 데이터 불일치를 잡는다.
