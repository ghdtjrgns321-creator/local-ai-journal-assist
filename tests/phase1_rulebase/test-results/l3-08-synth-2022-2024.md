# L3-08 Synthetic Evaluation — 2022~2024

> Rule: `MissingOrCorruptedDescription`  
> Dataset: `data/journal/primary/datasynth/journal_entries_{2022,2023,2024}.csv`  
> Labels: `data/journal/primary/datasynth/labels/anomaly_labels.csv`

## Rule Definition

Current Phase 1 L3-08 only flags explicit description loss/corruption:

- `missing`: `line_text + header_text` is blank or null
- `corrupted`: obvious placeholder/garbage such as `.`, `x`, `n/a`, `null`, special-only strings, jamo-only strings, or repeated characters/patterns

It no longer flags risk keywords such as `상품권`, `가계정`, `선급금`, `임시`, `기타`.

## Interpretation

- The all-label recall drop is expected: the old synthetic `VagueDescription` label mostly represents risk-keyword replacement, which is now Phase 3/NLP territory rather than Phase 1 L3-08.
- The apparent FP count is mostly unlabeled organic missing line/header descriptions in synthetic data. Under current label files, these count as FP even though they match the new Phase 1 rule definition.
- The remaining FN in the corrupt subset indicates the synthetic label changed header+lines in metadata, but not every labeled document surfaces as row-level `missing/corrupted` after the CSV is read. This needs sample-level lineage inspection before treating it as a detector bug.
