## Judgment

- Misses: none. L3-05 catches all 31 injected `WeekendPosting` documents across 2022~2024.
- Label/data validity: good for this rule. The injected `WeekendPosting` labels all land on Saturday or Sunday.
- False positives: high if `WeekendPosting` labels are treated as the only ground truth. This is mainly a scope mismatch, because L3-05 is implemented as a broad review signal for weekend or holiday postings, while the labels only mark injected weekend anomalies.
- Practical interpretation: L3-05 has perfect recall on injected labels but very low label precision. That does not necessarily mean the rule is broken; it means ordinary weekend/holiday activity exists in the synthetic population and is intentionally review-flagged by the current rule.

## Recommendation

If L3-05 should be evaluated as an injected-anomaly detector, add negative controls or a stricter rule variant, for example manual weekend postings only, non-system sources only, unusual user/process combinations, or weekend postings outside known allowed operations.

If L3-05 should remain a broad audit review signal, keep the current implementation but do not score its raw false positives against only `WeekendPosting` labels. Track it as `review_needed` with volume/triage metrics instead of confirmed-error precision.
