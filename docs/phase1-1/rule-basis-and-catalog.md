# PHASE1-1 룰 생성 기준과 룰 카탈로그

작성일: 2026-06-17

이 문서는 PHASE1-1 룰이 어떤 기준으로 만들어졌고, 각 룰이 무엇을 보는지, 현재 tier 체계에서 어떤 역할을 하는지 설명한다. 상세 발화조건과 예외는 [`docs/guide/룰원칙해설.md`](../guide/룰원칙해설.md)와 [`docs/spec/DETECTION_RULES.md`](../spec/DETECTION_RULES.md)를 따른다.

## 룰 생성 원칙

PHASE1-1 룰은 "정답 라벨을 잘 맞히는 feature"가 아니라 감사인이 전수 원장에서 설명 가능한 검토 후보를 만들기 위한 결정론적 기준이다.

| 원칙                  | 의미                                                                                                                      |
| --------------------- | ------------------------------------------------------------------------------------------------------------------------- |
| 명명 가능성           | 룰이 발화하면 감사인이 "무엇이 이상한지"를 룰명과 세부 근거로 설명할 수 있어야 한다.                                      |
| 전표/행 단위성        | PHASE1-1은 전표 또는 행 단위에서 볼 수 있는 조건을 맡는다. 계정/월 모집단, 그래프, 관계망, 시계열은 PHASE1-2로 보낸다.    |
| 근거 우선             | 금감원 감리지적 사례, ISA 240, PCAOB AS 2401, K-SOX/외감법, 일반 JE testing 절차에 닿는 조건을 우선한다.                  |
| 단독 신호와 맥락 분리 | 단독 review queue를 만들 수 있는 primary와, 다른 룰을 보강하는 booster/combo_only/macro_only를 분리한다.                  |
| 운영 언어 보수성      | 룰 발화는 후보, 예외, 검토 항목이지 fraud 확정이 아니다.                                                                  |
| 데이터 품질 분리      | 차대불균형, 필수필드 누락, 무효계정 같은 분석 가능성 문제는 fraud tier와 섞지 않는다.                                     |
| 테스트 끼워맞춤 금지  | DataSynth precision/recall은 개발 보조 지표다. 룰은 테스트 라벨을 맞히기보다 감사적으로 설명 가능한 원인을 유지해야 한다. |

## 29개 canonical 룰과 macro 중화

현재 canonical PHASE1-1 transaction rule은 29개다(count SoT: [docs/spec/RULE_DETAIL_METADATA_V1_LOCK.md](../spec/RULE_DETAIL_METADATA_V1_LOCK.md)). `L4-02/Benford`는 전표 단위 룰이 아니라 모집단 digit distribution finding이므로 canonical에서 제외된다. `L3-01`(계정 분류 불일치)과 `L3-08`(적요 결손/파손)은 2026-06-20 폐기되어 canonical 29개에서 제외된다.

Registry에는 `L4-02`, `Benford`, `D01`, `D02`가 남아 있다. 이유는 삭제 시 unknown rule fallback으로 primary 점수가 붙을 수 있기 때문이다. 따라서 이들은 `macro_only`, `role_factor=0`, `standalone_rankable=False`로 남겨 PHASE1-1 점수와 tier를 올리지 않게 중화한다.

## Role 용어

| role         | 의미                                             | 단독 tier             |
| ------------ | ------------------------------------------------ | --------------------- |
| `primary`    | 단독으로 topic seed가 될 수 있는 룰              | 가능. 조합 없으면 LOW |
| `booster`    | 다른 primary가 있을 때 맥락을 보강하는 룰        | 불가                  |
| `combo_only` | 특정 조합 트리거 재료로만 쓰는 룰                | 불가                  |
| `macro_only` | 모집단/계정/월 단위 finding. PHASE1-1에서는 중화 | 불가                  |

## Topic

| topic                 | 한글명                     | 역할                                                |
| --------------------- | -------------------------- | --------------------------------------------------- |
| `ledger_integrity`    | 원장기록·데이터정합성      | 분석 가능성, 원장 품질, 일부 약한 문서 품질 맥락    |
| `approval_control`    | 승인·권한·업무분장 통제    | 승인한도, 자기승인, 승인생략, SoD, 수기 입력        |
| `closing_timing`      | 결산·기간귀속·입력시점     | 결산기, cutoff, 전기일/문서일 괴리, 비업무시간 맥락 |
| `account_logic`       | 계정분류·거래실질 불일치   | 무효/오분류 계정, 비용자산화, 관계사 맥락, 가수금   |
| `duplicate_outflow`   | 중복·상계·자금유출         | 중복지급, 중복전표, 역분개, 한도 직하               |
| `revenue_statistical` | 수익·금액·모집단 통계 이상 | 매출 급변, 이상 고액, 배치성 전표                   |

## L1: 구조·통제 위반

| 룰                  | 설명                                                                           | 최신 역할                                                                                        |
| ------------------- | ------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------ |
| L1-01 차대변 불균형 | 전표의 차변/대변 합계가 맞지 않는 구조 오류를 잡는다.                          | 데이터 품질 게이트. fraud tier로 해석하지 않는다.                                                |
| L1-02 필수필드 누락 | 분석에 필요한 핵심 필드가 비어 있거나 파손된 경우를 잡는다.                    | 데이터 품질 게이트. 다른 룰의 미탐/과탐 원인으로 관리한다.                                       |
| L1-03 무효 계정     | CoA 밖 계정, placeholder, 깨진 계정코드 등을 잡는다.                           | 데이터 품질/계정논리 경계. 운영상 품질 트랙으로 분리한다.                                        |
| L1-04 승인한도 초과 | 승인 대상 금액이 승인자 한도를 넘거나 승인권 없는 사람이 승인한 경우를 잡는다. | primary. `critical`/`non_approver`는 HIGH floor. 승인우회 combo 재료.                            |
| L1-05 자기 승인     | 작성자와 승인자가 같은 사람인 전표를 잡는다.                                   | primary. 승인우회, 횡령은폐, 가공전표 2차정황 combo 재료.                                        |
| L1-06 직무분리 위반 | 금지된 업무 조합이 직접 확인된 SoD 충돌을 잡는다.                              | primary. 승인우회 combo 재료. 단순 업무범위 넓음은 L3-12가 담당한다.                             |
| L1-07 승인 생략     | 승인자가 없거나 승인 workflow가 빠진 전표를 잡는다.                            | primary. 승인우회, duplicate_outflow combo 재료.                                                 |
| L1-08 기간 불일치   | 회계기간과 전기일/회계연도가 맞지 않는 경우를 잡는다.                          | primary. closing_timing seed 및 timing combo 재료.                                               |
| L1-09 승인일 누락   | 승인자 자체가 아니라 승인일/승인 추적성이 비어 있는 경우를 잡는다.             | primary. MEDIUM combo 재료지만 근거가 약하고 과발화 이력이 있어 가공전표 2차정황에서는 제외됐다. |

## L2: 강한 통제우회·자금유출 신호

| 룰                  | 설명                                                            | 최신 역할                                                                                       |
| ------------------- | --------------------------------------------------------------- | ----------------------------------------------------------------------------------------------- |
| L2-01 승인한도 직하 | 승인한도 바로 아래 금액의 반복 또는 분할 가능성을 잡는다.       | primary. 한도분할 MEDIUM combo 재료. 단독은 우회 확정이 아니다.                                 |
| L2-02 중복 지급     | 같은 참조/거래처/금액 기반의 중복 지급 후보를 잡는다.           | primary. `reference_match`는 MEDIUM floor. 횡령은폐 combo 재료.                                 |
| L2-03 중복 전표     | 동일하거나 매우 유사한 전표 반복, 상계/중복 패턴을 잡는다.      | primary. 횡령은폐 및 가공전표 2차정황 combo 재료. 내부 alias `L2-03a~d`는 L2-03 드릴다운이다.   |
| L2-04 비용 자산화   | 비용으로 처리해야 할 성격을 자산으로 분류한 조작 후보를 잡는다. | primary. 2026-06-17 코드 기준 `L2-04 + L3-02 + L3-04`가 expense_capitalization HIGH로 발화한다. |
| L2-05 역분개 패턴   | 취소, 환입, 상계처럼 원거래를 되돌리는 전표 흐름을 잡는다.      | primary. 횡령은폐, 관련자 역분개, duplicate_outflow combo 재료.                                 |

| 룰                            | 설명                                                                | 최신 역할                                                                                   |
| ----------------------------- | ------------------------------------------------------------------- | ------------------------------------------------------------------------------------------- |
| L3-02 수기 전표               | manual/adjustment source 등 사람이 직접 넣은 JE 모집단을 잡는다.    | primary. 가공전표, 승인우회, 횡령은폐, 비용자산화의 핵심 combo 재료.                        |

| L3-04 기말/기초 결산 검토     | 결산기, 기말/기초 집중 전표를 잡는다.                               | primary. closing_timing seed, 관련자 역분개, 비용자산화 combo 재료.                         |
| L3-05 주말/공휴일 전기        | 비영업일에 전기된 전표를 잡는다.                                    | booster. 단독 queue 불가. 승인우회 MEDIUM 맥락.                                             |
| L3-06 심야 전기               | 비업무시간 또는 심야 입력 전표를 잡는다.                            | booster. 단독 queue 불가. 승인우회 HIGH/MEDIUM 맥락.                                        |
| L3-07 전기일-문서일 장기 괴리 | 전기일과 문서일 차이가 큰 backdating/cutoff 후보를 잡는다.          | primary. closing_timing seed와 timing combo 재료.                                           |
| L3-09 가수금 장기체류         | suspense/clearing/temporary 계정에 오래 남은 금액을 잡는다.         | primary. `L3-09 + outflow + L4-03` suspense concealment HIGH 재료.                          |
| L3-10 고위험 계정 사용        | 민감 계정, 감사인이 지정한 고위험 계정 사용을 잡는다.               | booster. 가공전표 2차정황과 timing weak context. 단독 queue 불가.                           |
| L3-11 매출 cutoff 불일치      | 기간귀속, cutoff, 반복/마감 전후 매출 인식 이상을 잡는다.           | primary. closing_timing HIGH 및 가공전표 2차정황.                                           |
| L3-12 업무범위 집중           | 특정 사용자가 비정상적으로 넓은 업무 범위를 처리하는 현상을 잡는다. | combo_only. L1-06과 달리 확정 SoD 위반이 아니라 review-only 맥락이다.                       |

| L4-06 배치성 자동 전표 검토 | 배치성, 반복성, 자동 전표 맥락의 이상을 잡는다.                       | combo_only. batch_combo, 가공전표 MEDIUM 재료.                                                                                                                                                                                      |

| 신호                     | 현재 위치                  | 이유                                                              |
| ------------------------ | -------------------------- | ----------------------------------------------------------------- |
| Benford/L4-02            | PHASE1-2 macro/family      | 전표 단건을 위반으로 설명하지 못하고 계정/모집단 분포를 설명한다. |
| D01 계정 활동량 급변     | PHASE1-2 timeseries/macro  | 전기 대비 계정 그룹 단위 분석이다.                                |
| D02 월별 분포 패턴 변화  | PHASE1-2 timeseries/macro  | 월/계정 단위 시계열 패턴이다.                                     |
| IC01~03 관계사 대사      | PHASE1-2 relational/family | 전표 단건이 아니라 거래쌍/회사 간 대사 문제다.                    |
| GR01 순환거래            | PHASE1-2 graph/family      | A→B→C→A 같은 연결 구조가 필요하다.                                |
| VAE/Isolation/ML anomaly | PHASE2                     | 알려진 룰명이 아니라 정상 분포 밖 companion score다.              |

## HIGH 자격이나 현재 룰이 직접 못 잡는 scheme

최신 [`HIGH_COMBO_GROUNDING.md`](../spec/HIGH_COMBO_GROUNDING.md)는 "우리 룰로 0건 잡혔으니 LOW"라는 자기참조 판단을 폐기했다. 등급은 감사기준, 금감원 실증, 외부 사례 3축으로 먼저 판정하고, 코드가 못 잡는 것은 탐지갭으로 분리한다.

| scheme                                      | tier 자격 | 현재 PHASE1-1 상태                                                                                                                                                                             |
| ------------------------------------------- | --------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 가공거래처(페이퍼·차명회사)                 | HIGH      | **GL-only 범위 외** — 거래처 마스터 비보유라 PHASE1-1 primary 룰 신설 안 함([CONSTRAINTS.md](../spec/CONSTRAINTS.md)). PHASE1-2가 원장 첫 등장·희소 약근사로 일부 surface(HIGH-6 커버 비주장). |
| 재고 과대평가(조작형)                       | HIGH      | **GL-only 범위 외** — 재고 보조원장(수량·단가·NRV) 비보유, 룰 신설 안 함([CONSTRAINTS.md](../spec/CONSTRAINTS.md) §재고). L3-10 계정코드만으로 대체하지 않는다. 등급 HIGH 유지.                |
| topside/연결조정 전표                       | HIGH 자격 | **GL-only 범위 외** — 연결 산출물(eliminations)이 PHASE1 입력 밖, 룰 신설 안 함([CONSTRAINTS.md](../spec/CONSTRAINTS.md) §연결조정). 등급 HIGH 유지.                                           |
| 휴면계정 활성화                             | LOW       | 회계분개 부정 근거가 약해 LOW scheme으로 둔다.                                                                                                                                                 |
| 단순 추정 누락, 추정형 재고 평가손실 미인식 | LOW       | 위조·고의·강한 전표 직접조작 신호가 없으면 LOW다. 조작형 재고 과대평가와 구분한다.                                                                                                             |
