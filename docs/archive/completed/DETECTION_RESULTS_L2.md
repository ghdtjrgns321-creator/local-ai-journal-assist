# Detection Results

> **PHASE1 역할 원칙**: PHASE1은 `fraud`를 확정하거나 정답 라벨을 맞히는 단계가 아니다. PHASE1의 목적은 전수 모집단에서 규칙 위반, 정책 위반, 이상 징후, 분석적 검토 신호를 넓게 올려 **감사인이 봐야 할 항목과 우선순위**를 만드는 것이다. DataSynth의 `is_fraud`/`is_anomaly`와 precision/recall은 개발 검증 보조 지표이며, 운영 해석은 예외 처리 대상, 감사인 리뷰 대상, 고위험 후보를 구분하는 review queue 기준으로 한다.

## Phase 1 L2 - DataSynth v126

대상 데이터는 `data/journal/primary/datasynth_v126_candidate`입니다.

이 문서는 L2 결과를 **세 가지 독립적인 검증 축**으로 나누어 봅니다. 세 축은 측정하는 대상이 다르므로 같은 표에 섞으면 의미를 잃습니다.

| 축  | 무엇을 검증하는가                                                  | 검증 데이터                                                        | 합격 기준                                                   |
| :-- | :----------------------------------------------------------------- | :----------------------------------------------------------------- | :---------------------------------------------------------- |
| A   | 데이터 정합성 / 룰 계약사항이 문서 단위로 동작하는가                  | `rule_truth_L2_*.csv` (공식 정답)                                   | 정탐 100%, **과탐 0, 미탐 0**                               |
| B   | DETECTOR를 직접 참조하지 않고 "실제로 있을법한" 케이스를 잡아내는가 | L2 sidecar 파일 (`l201_just_below_threshold_truth.csv` 등)          | 책임/점수/검토/미포착 분리 타당성                           |
| C   | 실제로 악의적으로 조작된 전표를 책임 영역 안에서 잡아내는가           | `manipulated_entry_truth.csv`                                       | L2 책임 거래 구조 패턴 안에서 포착, 나머지는 L1~L4 통합 해석 |

세 축의 관계:

- A축은 detector 사양과 데이터 생성기가 같은 정의를 공유하는지 보는 **계약 검증**입니다. 0/0이 합격선입니다.
- B축은 **detector 로직을 모르는 입장**에서 "현장에서 실제로 나올 만한 형태"를 sidecar로 만든 뒤 detector가 어떻게 반응하는지 봅니다. L2는 후보를 먼저 넓게 잡고 점수로 분리하는 룰이 많으므로 "잡음 vs 점수 vs 검토" 분리가 핵심입니다.
- C축은 **fraud 시나리오 합성 데이터**가 L2 책임 영역(거래 구조 패턴)에 어떻게 걸리는지 봅니다. `manipulated_entry_truth.csv`는 L2 룰별 정답이 아니라 L1~L4 전체 신호 조합으로 surfaced 되는지를 보는 데이터입니다.

L2의 평가 방식은 룰별로 strict와 population으로 갈립니다. strict 룰(L2-01, L2-05)은 공식 정답을 정확히 맞춰야 하고, population 룰(L2-02, L2-03, L2-04)은 raw review universe를 누락 없이 잡은 뒤 high/medium/low/zero band로 분리합니다.

## 요약

- **A축 결과**: 3개 연도 x 5개 룰 기준 정답 2,130건 모두 정탐. 과탐 0, 미탐 0. 룰 계약 정합성 합격.
- **B축 결과**: v126 L2 sidecar 6개 모두 현재 후보 데이터 기준. L2-02/03/04는 후보를 모두 잡고 점수/검토로 분리하는 Phase1 기준과 일치.
- **C축 결과**: 조작 전표 420건 중 L2가 직접 surfaced 한 전표는 6건. L2 단독 미포착 414건은 L2 실패가 아니라 대부분 L1/L3/L4 책임 영역 또는 복합 신호 평가 대상입니다.

세 축을 분리하는 이유:

- A축만 통과: detector가 자기 정의대로 도는 것뿐. 점수 분리 현실성은 알 수 없음.
- A+B축까지 통과: "후보로 잡는 범위"와 "위험점수에 넣는 범위"가 분리되어 있는지 확인 가능.
- A+B+C축 통과: 계약, 현실 모집단, 실제 조작 데이터의 세 축이 모두 설명 가능.

또 "L2가 잡았다"와 "최종 위험점수에 강하게 반영한다"는 다릅니다. 예를 들어 L2-04는 자산+비용 조합을 넓게 잡지만 1,098건 중 1,020건은 zero band로 남기고 78건만 high band로 올립니다. 이 분기 자체가 B축 영역입니다.

## 검증 축 B — Sidecar 현실 케이스 포착력 (DETECTOR 비참조)

이 축은 **detector 코드를 직접 참조하지 않고** "현장에서 실제로 나올 만한 케이스"를 sidecar로 만들어 둔 뒤 detector가 그것을 어떻게 분기하는지 봅니다. L2는 Phase1 전체 점수에서 거래 구조 패턴 신호이므로 B축의 핵심은 "넓게 잡은 후보를 위험점수에 직접 넣을지 검토 후보로 뺄지"의 분기가 detector 동작과 정합한지입니다.

### Sidecar 평가 기준

| 항목      | 의미                                                                       |
| :-------- | :------------------------------------------------------------------------- |
| L2 책임   | 해당 sidecar가 L2가 직접 잡아야 하는 대상인지 여부                         |
| 실제 잡음 | L2 detector가 실제로 포착한 단위 수                                        |
| 점수      | 최종 위험점수에 의미 있게 반영해야 하는 단위 수                            |
| 검토 별도 | 잡히지만 위험점수는 약하게 두거나 0점으로 두는 후보/별도 설명 단위 수       |
| 미포착    | 해당 L2 룰이 잡지 못한 단위 수                                             |

L2 책임 값의 의미:

| 값    | 의미                                                                  |
| :---- | :-------------------------------------------------------------------- |
| 예    | L2가 직접 잡아야 하는 대상. 잡으면 L2 점수 또는 검토 신호에 반영       |
| 별도  | 공식 truth의 보조 sidecar. 같은 모집단을 다른 관점에서 확인           |

## 검증 축 C — 악의적 조작 데이터 포착력 (`manipulated_entry_truth.csv`)

`manipulated_entry_truth.csv`는 **fraud 시나리오를 의도적으로 합성한 전체 조작 전표 표본**입니다. L2 특정 룰의 정답이 아니므로 L2가 전부 직접 위험점수로 잡아야 하는 대상은 아닙니다. L2는 중복 지급, 중복 전표, 비용 자본화, 역분개, 승인한도 근접 같은 거래 구조 패턴이 겹치는 조작 전표만 직접 surfaced 합니다.

### 전체 요약

| 항목                    | 건수 |
| :---------------------- | ---: |
| 전체 조작 전표          |  420 |
| L2가 실제 잡은 전표     |    6 |
| L2가 잡지 않은 조작 전표 |  414 |

v126 기준으로 전체 조작 전표 420건 중 6건이 L2 후보 또는 점수 신호로 surfaced 됩니다. 나머지 414건은 대부분 L1 승인/직무분리/기간 통제, L3 행태/관계/시점 신호, L4 사용자/네트워크 신호와 결합해서 봐야 합니다.
