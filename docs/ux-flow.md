# UX 데이터 흐름

> **PHASE1 역할 원칙**: PHASE1은 `fraud`를 확정하거나 정답 라벨을 맞히는 단계가 아니다. PHASE1의 목적은 전수 모집단에서 규칙 위반, 정책 위반, 이상 징후, 분석적 검토 신호를 넓게 올려 **감사인이 봐야 할 항목과 우선순위**를 만드는 것이다. DataSynth의 `is_fraud`/`is_anomaly`와 precision/recall은 개발 검증 보조 지표이며, 운영 해석은 예외 처리 대상, 감사인 리뷰 대상, 고위험 후보를 구분하는 review queue 기준으로 한다.

> 작성 기준: 2026-04-16  
> 이 문서는 현재 제품 UX의 기준 문서다. 핵심 목표는 사용자가 "지금 이 시스템의 메인 탐지 경로가 무엇인지"를 헷갈리지 않게 만드는 것이다.

---

## 1. UX 핵심 원칙

### 원칙 1. 첫 메시지는 단순해야 한다

앱에 들어오자마자 사용자는 아래 한 줄을 이해해야 한다.

> 먼저 감사 룰로 설명 가능한 이상을 찾고, 그다음 비지도 탐지로 룰 밖 패턴을 보완한다.

### 원칙 2. 탐지기는 병렬 소개가 아니라 위계로 소개한다

- `Core`: Phase 1 룰 기반, Phase 2 비지도
- `Optional`: 지도학습
- `Experimental`: 스태킹, 고급 모델
- `Future`: NLP, 그래프, LLM 판단

### 원칙 3. 보조 기능은 탐지 뒤에 나온다

- Chat
- 룰 제안
- Export

이 기능들은 중요하지만, 핵심 탐지 축보다 앞에 나오면 안 된다.

---

## 2. 사용자 흐름

```text
회사 선택
-> Engagement 선택
-> 파일 업로드 / 배치 복원
-> 입력 확인
-> 데이터 품질 확인
-> Phase 1 룰 기반 탐지 결과 확인
-> Phase 2 비지도 보완 확인
-> 전표 탐색
-> 보조 기능 활용
```

기존 UX 문서에서 강조하던 `신규 회사 vs 기존 회사` 구분은 내부 구조상 여전히 중요하다.  
다만 사용자 메시지의 첫 줄은 그보다 `핵심 탐지 경로`가 먼저여야 한다.

---

## 3. 화면 구조

### 3.1 상단 헤더

헤더는 아래 내용을 짧게 보여준다.

- 현재 배치 이름
- 총 분석 행 수
- 핵심 탐지 축 요약

예시 메시지:

> 핵심 경로는 Phase 1 룰 기반 탐지와 Phase 2 비지도 보완 탐지입니다.

### 3.2 사이드바

사이드바는 아래 순서가 맞다.

1. 회사/Engagement 정보
2. 데이터 필터
3. 핵심 탐지 기준
4. 재탐지 실행

세부 threshold 조정은 기본 노출이 아니라 expander 안에 둔다.

### 3.3 메인 탭

메인 탭은 아래 구조를 따른다.

1. `핵심 탐지`
2. `데이터 품질`
3. `전표 탐색`
4. `비지도 보완`
5. `Chat`
6. `룰 제안`
7. `Export`

여기서 중요한 점은 `비지도 보완` 탭이 "ML 전체 소개"가 아니라 "Phase 2 메인 확장 축 설명" 탭이어야 한다는 것이다.

---

## 4. 단계별 UX

### 4.1 업로드 단계

사용자가 이 단계에서 이해해야 하는 것은 "내 파일이 탐지 가능한 상태인가"이다.

따라서 UI 메시지도 아래에 맞춘다.

- 헤더 탐지
- 컬럼 매핑
- 타입 정리
- 필수 컬럼 부족 경고

이 단계는 "고급 매핑 기능 소개"가 아니라 "탐지 가능한 입력 준비" 단계다.

### 4.2 데이터 품질 단계

이 탭은 탐지 전에 데이터 신뢰도를 확인하는 단계다.

- 결측
- 중복
- 분포 이상
- 기본 EDA

사용자 메시지는 "좋은 데이터인가"가 아니라 "이 데이터로 탐지 결과를 얼마나 믿어도 되는가"에 맞춰야 한다.

### 4.3 핵심 탐지 단계

이 탭은 프로젝트에서 가장 중요한 탭이다.

- KPI
- 위험도 분포
- 룰 위반 분포
- 월별 집중 구간

그리고 항상 아래 순서를 유지한다.

1. 설명 가능한 룰 기반 결과
2. 그 결과를 기반으로 한 우선순위

### 4.4 비지도 보완 단계

이 탭은 Phase 2를 과장하지 않는다.

- 현재 메인 확장 축은 비지도 탐지임을 설명
- 왜 지도학습이 기본값이 아닌지 설명
- 실행된 탐지 트랙이 있다면 표로 보조 노출

즉 "모든 ML 모델 자랑"이 아니라 "왜 비지도가 메인인지 설명"하는 탭이다.

### 4.5 전표 탐색 단계

탐색 화면은 결과 해석의 실행 단계다.

- 필터된 전표 목록
- 상세 근거
- HITL 예외 처리

탐지기의 종류를 여기서 다시 장황하게 보여줄 필요는 없다.  
이미 앞 탭에서 해석 축을 이해한 뒤 전표 수준으로 내려오는 구조가 맞다.

## 5. 상태 문구 기준

진행률이나 상태 문구도 새 기준을 따라야 한다.

좋은 예:

- `회사별 배치 데이터를 조회하는 중...`
- `핵심 탐지 결과를 복원하는 중...`
- `감사 룰 평가 중...`
- `정상 분포 이탈을 보완 분석하는 중...`

피해야 할 예:

- `ML/DL 모델 실행 중`
- `VAE, XGBoost, 앙상블 실행 중`

이런 표현은 메인과 보조를 같은 무게로 보이게 만든다.

---

## 6. 설정 UX 기준

설정은 많을수록 좋은 것이 아니다.

### 기본 노출

- 룰 기반 threshold
- 핵심 preset
- 재탐지 버튼

### 접어둘 항목

- 세부 threshold
- 실험적 옵션
- 고급 모델 관련 설명

즉 UX는 "처음부터 다 보여주는 것"보다 "핵심 경로를 먼저 이해시키고 필요한 사람만 깊게 들어가게 하는 것"이 맞다.

---

## 7. UX 한 줄 요약

이 제품의 UX는 `기능 나열형 UX`가 아니라 `핵심 탐지 축을 먼저 확신시키는 UX`다.

따라서 모든 화면은 아래 질문에 답해야 한다.

> 사용자가 지금 이 시스템의 메인 탐지 경로를 바로 이해할 수 있는가?
