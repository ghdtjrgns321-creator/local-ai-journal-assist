# PHASE1 Tier Evidence Basis — 근거 기반 tier 체계

> **PHASE1 역할 원칙**: PHASE1은 `fraud`를 확정하거나 정답 라벨을 맞히는 단계가 아니다. PHASE1의 목적은 전수 모집단에서 규칙 위반, 정책 위반, 이상 징후, 분석적 검토 신호를 넓게 올려 **감사인이 봐야 할 항목과 우선순위**를 만드는 것이다. DataSynth의 `is_fraud`/`is_anomaly`와 precision/recall은 개발 검증 보조 지표이며, 운영 해석은 예외 처리 대상, 감사인 리뷰 대상, 고위험 후보를 구분하는 review queue 기준으로 한다.

> **포트폴리오 주장 범위**: 이 프로젝트는 `fraud`를 판정하거나 실제 운영 부정 탐지 성능을 보장하는 모델이 아니다. 전수 모집단에서 감사인이 먼저 볼 review queue를 만들고, 무작위 검토 대비 상위 구간에 review-worthy synthetic anomaly를 강하게 농축하는 로컬 감사 분석 보조 도구다. DataSynth 기반 precision/recall은 개발 검증 보조 지표이며, 실데이터 운영 성능으로 주장하지 않는다.
> **금지 표현**: "부정을 정확히 탐지", "실무 운영 성능 검증 완료", "TOP100 precision 충분", "fraud 확정/자동 적발"처럼 확정적이거나 운영 성능을 보장하는 표현은 사용하지 않는다.

## 1. 이 문서의 목적

PHASE1 case 우선순위(band)를 **근거 있는 tier 체계**로 재정의하기 위한 근거 출처 문서다.

기존 점수체계는 가중합(`0.62·max_primary + 0.08·secondary + …`)과 floor 숫자(`0.75`/`0.60`/`0.45`), band 컷(`high≥0.90`/`medium≥0.75`)으로 case를 줄세웠다. 분석 결과 두 가지 문제가 확인됐다.

1. **가중합 계수는 근거가 없다.** 0.62/0.08 등은 설계자가 손으로 정한 값이고, PHASE1은 정책상 truth recall로 가중치를 보정하는 것이 금지(`feedback_phase1_truth_recall_guard`)되어 데이터로 정당화할 길도 없다.
2. **band는 사실 가중합이 아니라 floor/combo가 만든다.** 가중합 단독으로는 high(0.90)에 도달하지 못한다(전 컴포넌트 동시 만점 필요). 실제 high/medium은 floor·combo floor가 결정한다 — `docs/archive/completed/PHASE1_TOPIC_SCORING_V1_LOCK.md`(2026-06-16 archive 이관) line 106이 이미 자인한 사실이다.

따라서 tier 체계는 **숫자 장난(floor 숫자 → band 컷 비교)을 제거**하고, 근거 있는 조건에서 tier로 직접 매핑한다. 이 문서는 그 조건의 근거(금감원 감리 사례·ISA 240/PCAOB AS 2401·K-SOX·실제 위반)를 정리한다.

근거 출처(이미 정제된 본 프로젝트 문서):

- [`DETECTION_REFERENCE.md`](DETECTION_REFERENCE.md) §2 (기준서 → 룰 도출), §3 (금감원 감리 189건 6대 패턴 → 룰 매핑)
- [`PHASE1_RULE_RELATIONSHIP_MAP.md`](PHASE1_RULE_RELATIONSHIP_MAP.md) §7 (시나리오별 증폭 조합 + PCAOB AS 대조)
- [`PHASE1_TOPIC_SCORING_V1_LOCK.md`](../archive/completed/PHASE1_TOPIC_SCORING_V1_LOCK.md) §Fraud Combo Scoring Policy (조합 → floor + 금감원 건수) — 2026-06-16 archive 이관
- 원본 금감원 감리 사례: `data/finding/` (FSS 189개 PDF·HWP)

## 2. PHASE1의 역할 — tier는 "판정"이 아니라 "큐 우선순위"

**HIGH / MEDIUM / LOW는 위반을 "찾았다/판정했다"는 등급이 아니다.** 감사인이 한정된 시간에 전수 모집단을 다 볼 수 없으므로, **무엇을 먼저 볼지** 정하는 순서다.

| tier    | 의미 (감사인 관점)                                                   | "판정"이 아님                            |
| ------- | -------------------------------------------------------------------- | ---------------------------------------- |
| HIGH    | 먼저 봐야 할 항목 — 감사기준이 journal entry risk로 명시한 조합 발화 | fraud 확정 아님. 검토 우선순위 승격일 뿐 |
| MEDIUM  | 그다음 볼 항목 — 도메인상 위험하나 단일 축이 부족                    | 위반 확정 아님                           |
| LOW     | 큐에는 올라오되 우선순위 낮음 — 단일 규칙 위반/이상 신호             | 정상일 가능성 높음, 참고용               |
| CONTEXT | 단독으로 큐를 만들지 못하는 보조/모집단 신호 (macro/booster)         | 다른 항목의 맥락 보강                    |

PHASE1이 못 잡는 잔여(룰로 표현되지 않는 패턴)는 PHASE2 ML이 담당한다(§7). 즉 tier는 "이 전표는 부정이다"가 아니라 "감사인이 이 순서로 보면 효율적이다"라는 **review queue 정렬 기준**이다.

## 3. 설계 원칙 — 근거 있는 것과 없는 것을 분리한다

기존 floor 구조에는 근거가 있는 부분과 없는 부분이 섞여 있었다. tier 체계는 이를 분리한다.

| 요소                                      | 근거                                                 | tier 체계에서의 처리                                                                 |
| ----------------------------------------- | ---------------------------------------------------- | ------------------------------------------------------------------------------------ |
| **트리거 조건** (어떤 룰 조합이 위험한가) | **있음** — 금감원 사례수·ISA 240·PCAOB AS 2401·K-SOX | tier 결정의 유일한 근거. 이 문서 §4~§6이 전부 명시                                   |
| **floor 숫자값** (0.75 vs 0.60)           | **없음** — 임의 숫자                                 | 제거. 조건 → tier 직접 매핑, 숫자 안 거침                                            |
| **band 컷** (high=0.90)                   | **없음** — 임의 숫자                                 | 제거. tier는 순서형(HIGH>MED>LOW), 크기 의미 없음                                    |
| **tier 내부 정렬** (가까운 case 줄세우기) | 약함                                                 | 단순·방어가능 키로만(예: 독립 primary 룰 수, materiality). 확률/위험도라 부르지 않음 |

**핵심**: tier는 **순서형(ordinal)**이다. HIGH가 MEDIUM보다 "먼저 본다"는 순서일 뿐, "0.9만큼 위험"이라는 크기(cardinal)가 아니다. 두 HIGH case 사이의 미세 점수차는 의미를 부여하지 않는다(false precision 회피).

## 4. tier 결정 조합 — 4개 시나리오 패밀리

금감원 감리 189건 중 전표 관련 94건은 6대 패턴으로 분류됐다(`DETECTION_REFERENCE.md` §3.3). 그중 순환거래는 PHASE2로 이관(§7), 비정상시점은 결산·시점 booster로 흡수되어, **PHASE1 tier를 만드는 패밀리는 4개**다.

```
가공전표 (FSS 50건, 53%)  → 수익·금액·모집단 통계 이상
결산수정 (FSS 27건, 29%)  → 결산·기간귀속·입력시점
횡령은폐 (FSS 24건, 26%)  → 중복·상계·자금유출
승인우회 (FSS  5건,  5%)  → 승인·권한·업무분장 통제
```

각 패밀리는 HIGH 조건과 MEDIUM 조건을 가진다. 룰 id는 [`DETECTION_RULES.md`](DETECTION_RULES.md) 참조.

### 4.1 가공전표 의심 → 수익·금액·모집단 통계 이상

> 근거: FSS 189건 중 최다(50건, 53%). 실물 없이 세금계산서/계약서 위조 후 매출/자산 분개 생성. ISA 240 A45와 PCAOB AS 2401 §61이 수기/조정 전표·희소 계정·비정상 계정·설명 부족·금액 분포 이상을 journal entry risk 특성으로 명시.

| tier   | 트리거 조건                                                                                                                                                                                                                 | 근거                                            |
| ------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------- |
| HIGH   | `(L4-01 매출이상 or L4-03 이상고액) + L3-02 수기 + 2차정황1개` <br>2차정황 = `L4-04 희소계정쌍 / L2-03 중복 / L3-03 관계사 / L3-10 민감계정 / L1-05 자기승인 / L3-11 cutoff` 중 하나 (L3-04·L1-09는 과탐 가드로 제외, §4.5) | ISA 240 A45(a)(b)(e) + A안 셋째 다리 확장(§4.5) |
| MEDIUM | `L4-01 + L3-04 결산` 또는 `L4-03 + L4-06 배치 + L3-02 수기`                                                                                                                                                                 | 금액 이상 + 결산/배치 맥락                      |

> **A안 셋째 다리 확장 (2026-06-16, §4.5)**: 위 HIGH 조합의 2차정황 풀은 원래 `(L4-04 or L2-03)` 2종뿐이었다. FSS 실증 17건 재감사 결과, 같은 가공전표 스토리인데 셋째 다리만 다른 11건(관계사 경유·기말·자기승인·cutoff 등)이 그물을 빠져나갔다. 스토리가 동일하므로 신규 조합을 만들지 않고 인정 2차정황을 FSS 실증에 맞게 넓혔다. 상세 감사·근거는 [`HIGH_COMBO_GROUNDING.md`](HIGH_COMBO_GROUNDING.md) §5·§5b.

### 4.2 결산수정 조작 의심 → 결산·기간귀속·입력시점

> 근거: FSS 27건(29%). 손상 미인식·충당금 환입·원가 이연·개발비 과대자산화. ISA 240 §32(a)(ii) "보고기간 말 전표 선정·검사" 직접 명시, PCAOB AS 2401 §61 기말/마감후 수정·설명 부족.

| tier | 트리거 조건                                                                                                         | 근거                                   |
| ---- | ------------------------------------------------------------------------------------------------------------------- | -------------------------------------- |
| HIGH | `시점seed(L3-04 결산/L3-07 전기일괴리/L3-11 cutoff/L1-08 기간불일치) + L4-03 고액 + (L3-10 민감계정 or L4-04 희소)` | ISA 240 A45(c) + 고액 + 설명/계정 이상 |
| HIGH | `L3-11 cutoff + (L4-01 or L4-03)`                                                                                   | cutoff 불일치 + 고액/매출              |

### 4.3 횡령은폐 의심 → 중복·상계·자금유출

> 근거: FSS 24건(26%, 제목만으론 6건이나 본문 읽기로 4배). 선급금/대여금/매출채권 허위계상으로 횡령액 은폐. 오스템임플란트(2021) 재무팀장 1인 입력·승인·이체 전부 수행, 2,215억 횡령. K-SOX(외감법 §8①5호) 업무분장·승인 통제 실패.

| tier   | 트리거 조건                                                                               | 근거                                                                                      |
| ------ | ----------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------- |
| HIGH   | `자금유출(L2-02/L2-03/L2-05) + [승인우회(L1-04/05/06/07) or (L2-05 역분개 + L3-02 수기)]` | 자금유출 + 통제우회 = 횡령은폐 전형. 승인흔적 없는 은폐(역분개+수기) 분기 추가(§4.5, A안) |
| MEDIUM | `L2-01 한도직하 + (L1-04 or L1-05)`                                                       | 분할 + 승인우회                                                                           |

> **A안 — 조합2 승인룰 완화 (2026-06-16, §4.5)**: 원래 조합2는 자금유출 + 승인우회를 **둘 다 필수**로 요구했다. 그러나 FSS 감리2013-1-가·A-6유형-가는 명백한 횡령은폐인데 승인 신호가 안 뜨고 역분개(L2-05) + 수기(L3-02) + 고액으로만 나타나 탈락했다. 횡령은폐가 항상 승인 흔적을 남기지는 않으므로, 승인우회 OR (역분개+수기) 분기를 추가했다.

### 4.4 승인우회 조작 의심 → 승인·권한·업무분장 통제

> 근거: FSS 5건 + 횡령은폐의 전제 조건. K-SOX 내부회계관리제도 운영 효과성, 외감법 §8②(우회 금지), 감사기준서 315호 §26 비표준 전표 통제.

| tier   | 트리거 조건                                                                                     | 근거                             |
| ------ | ----------------------------------------------------------------------------------------------- | -------------------------------- |
| HIGH   | `승인우회(L1-04/05/06/07) + 강한맥락(L4-03 고액 or L3-11 cutoff or L3-04+L3-02 or L3-06+L3-02)` | 통제우회 + 고액/시점/수기 강결합 |
| MEDIUM | `L1-09 승인일누락 + L4-03 + L3-02`                                                              | 추적성 약화 + 고액 + 수기        |
| MEDIUM | `승인우회 + L3-02 수기` / `+ L3-06 심야` / `+ L3-05 휴일`                                       | 통제우회 + 단일 맥락             |
| MEDIUM | `L3-12 업무범위집중 + (L1-05 or L1-07)`                                                         | K-SOX SoD 집중                   |

### 4.5 A안 — 셋째 다리 확장 (2026-06-16 확정)

FSS HIGH 158건을 전수 재집계한 결과, 기존 5조합 밖에 `HIGH(NEW조합)`로 따로 태깅된 17건이 있었다. 17건 원문(수법)을 까서 "다른 스토리(=신규 조합)냐, 같은 스토리·셋째 다리만 다르냐"를 판정했다.

```
같은 스토리(셋째 다리만 다름)  11건  → 조합1/2의 2차정황 풀 확장으로 흡수
이미 기존 조합 커버(오태깅)     2건  → 태깅만 정정
family → PHASE1-2 관할         3건  → 순환거래(graph). PHASE1-1 조합 아님
2-rule 약신호                  1건  → HIGH 자격 의심(별도 검토)
```

결론: 11건 전부 조합1(가공전표) 또는 조합2(횡령은폐)의 **스토리 그대로**, 그 스토리를 거드는 2차 정황의 종류만 달랐다. 따라서 신규 조합을 만들지 않고(B/C안 기각) **기존 조합의 인정 2차정황을 FSS 실증에 맞게 넓히는 A안**을 채택했다(§4.1·§4.3 반영).

메타패턴으로 압축하면 HIGH는 결국 **`수기(L3-02) + 고액(L4-03/매출급변 L4-01) + 2차 비정상 1개`**이며, 이는 PCAOB AS 2401 §61("복수 위험특성이 겹치면 고위험")의 구현이다. 단 메타규칙을 그대로 단일 HIGH 규칙으로 일반화(B안)하지 않은 이유는 약한 2차정황까지 HIGH로 올라 과탐이 커지기 때문이다.

> **과탐 가드(2026-06-17 측정·반영 완료)**: 정상 데이터(v42j 2022, 14,070 case)로 HIGH 비율을 측정한 결과 wide(6 신규 다리)는 **5.245%**로 HARD 가드(HIGH ≤ 2%)를 초과했다. 다리별 분해에서 **기말(L3-04) 734/738·승인일공백(L1-09) 334**가 과발화 주범(정상 결산전표), L3-03·L3-10·L1-05는 정상 0 발화였다. 기말은 closing_timing 조합의 영역이고 L1-09는 근거 약한 룰(§2-2)이라 둘을 2차정황에서 제외 → narrowed **0.334% PASS**. 최종 2차정황 = `L4-04·L2-03·L3-03·L3-10·L1-05·L3-11`. 상세: [`HIGH_COMBO_GROUNDING.md`](HIGH_COMBO_GROUNDING.md) §5b.

| `L1-04` (critical / non_approver 라벨)     | HIGH                   | 승인한도 명백 초과 / K-SOX §8①5호                                              |
| `L2-02` (reference_match)                  | MEDIUM                 | 동일 참조 중복지급 / K-SOX §8①4호 위변조 방지                                  |
| `L1-01`/`L1-02` (차대불균형·핵심필드 누락) | (데이터무결성 blocker) | 외감법 §8①1·2호 오류통제. 원장 신뢰성 게이트 — fraud tier가 아니라 품질 게이트 |

> 데이터무결성(원장기록) 토픽은 fraud 승격 토픽이 아니라 **품질 게이트**다(`docs/archive/completed/PHASE1_TOPIC_SCORING_V1_LOCK.md`). 다른 조작 combo의 concealment/context booster로만 fraud 해석에 참여한다.

## 7. 탐지 아키텍처 — 3-surface 구조 (C안, 2026-06-14 확정)

탐지는 출력 의미(output semantics)가 다른 **3개 surface**로 나뉜다. 단층선은 "알려진/미지"가 아니라 **"근거 있는 결정론적 명명 탐지 vs 학습된 통계 surface"**다.

| surface             | 무엇으로                                                             | 출력 단위·형태                                                                    | 성격                                        | 정직한 주장                                                         |
| ------------------- | -------------------------------------------------------------------- | --------------------------------------------------------------------------------- | ------------------------------------------- | ------------------------------------------------------------------- |
| **PHASE1-1 룰**     | 전표/행 단위 규칙                                                    | 전표 1건 · 명명된 위반 + tier                                                     | 결정론·근거(금감원 4대패턴 16조합)·설명가능 | "감사표준 명문 패턴을 결정론적 큐로"                                |
| **PHASE1-2 family** | 그래프·관계·시계열 탐지기 + macro(Benford 모집단통계·D01/D02 시계열) | 그룹(순환 A→B→C→A, 직원-거래처 쌍, 시계열)·계정/월 모집단 · 명명된 구조/통계 패턴 | 결정론·근거(같은 금감원 corpus)·설명가능    | "한 전표로 안 보이는 알려진 구조/모집단을 전용 탐지기로"            |
| **PHASE2 VAE**      | 정상 분포 비지도 학습                                                | 전표(문서) · 점수 + 비정형 사유                                                   | 학습·통계·companion                         | "학습된 정상 밖 비정형을 추가 검토 후보로 surface (부정 확정 아님)" |

### 7.1 불변식 — 3 surface 절대 비병합

세 surface는 **출력 의미가 다른 종류**다. 룰·family는 "구체적 명명 발견", VAE는 "막연한 비정형 점수"다. 한 점수·한 리스트로 병합하면 단위·신뢰도가 안 맞아 표시가 깨진다(과거 family+VAE 병합 실패의 원인). **각자 독립 surface(독립 탭/뷰/큐)로 표시하며 단일 점수로 합치지 않는다.**

### 7.2 단층선과 명명 근거

- **PHASE1(1-1 + 1-2)**: 둘 다 결정론적·근거(금감원)·명명된 발견. 단위만 다름(전표 vs 구조). 같은 부류라 PHASE1로 묶되 표시는 1-1/1-2로 분리.
- **PHASE2(VAE)**: 학습·통계·companion. 셋 중 가장 약한 보조 신호라 혼자 격리한다. 격리가 곧 7.1 병합 방지책이다.
- PHASE3는 쓰지 않는다(구 LLM Review Narrator 자리 — active product path에서 제거됨).

### 7.3 순환거래의 집 (PHASE1 IC 제거의 귀결)

PHASE1-1에서 제거된 관계사·내부거래·순환거래(FSS 10건)는 **PHASE1-2 family의 정식 대상**이다. 한 전표만 봐선 안 보이고 여러 전표·관계를 엮어야 보이므로 룰(PHASE1-1)로 표현 불가 → graph/relational 탐지기(family)가 맡는다. 근거: IFRS 10 §B86, K-IFRS 1110/1024, ISA 600/550. L3-03은 PHASE1-1에서 account_logic booster(관계사 모집단 맥락)로만 잔존.

### 7.4 정직성 가드 (3 surface 공통)

- PHASE1-1·1-2: "알려진 패턴을 명명 탐지" — "미지 부정 적발" 표현 금지.
- PHASE2 VAE: "정상 밖 비정형 surface" — "이상치=부정" 표현 금지.
- 어느 surface도 "세상 부정의 X% 적발"을 주장하지 않는다(분모 부재). 검증은 "PHASE1 밖 추가분" 같은 셀 수 있는 분모로만.

이 분담은 한계의 인정이 아니라 아키텍처 판단이다. "룰은 단일 전표 알려진 패턴, family는 구조적 알려진 패턴, VAE는 명명 안 된 비정형"이 설계 의도다.

> 코드 정합(family 탐지기의 PHASE1-2 귀속, VAE의 PHASE2 단독 surface화)은 구현 단계 항목이다. 본 문서는 역할·명명 결정을 확정하고, 코드·기존 문서 정합은 §9 및 별도 surgery로 이월한다.
