# PHASE1-1 조합 빌더 스펙 — tier 폐지, 감사인 조합 선택 (2026-07-17 확정)

> **결정 (사용자 확정, 2026-07-17)**: PHASE1-1의 자동 등급 체계(HIGH/MEDIUM/LOW/CONTEXT tier, combo floor 12종,
> has_rankable_primary 게이트, 조합별 특례)를 **전면 폐지**하고 **조합 빌더 + 프리셋**으로 대체한다.
> 데이터정합성(L1-01~03)은 빌더와 무관하게 **별도 패널**로 표시한다.
> 본 문서가 PHASE1-1 검토 표면의 새 SoT다. 구 SoT [HIGH_COMBO_GROUNDING.md](HIGH_COMBO_GROUNDING.md)는 supersede.

## 1. 왜 폐지하나 (근거 요약)

1. **등급 근거의 자의성**: 구 체계의 HIGH/MEDIUM 판정은 (a) 감사기준 인용(방향만 제공) (b) 금감원 실증 건수
   (v1 태깅 기반 — v3 엄격 재태깅에서 붕괴) (c) 조합별 전문가 특례("보강신호 1개" 등)로 구성됐고,
   실질 결정력은 (c)였다. 조합마다 다른 사연 = 방어 불가 휴리스틱.
2. **FSS 텍스트의 구조적 한계**: 금감원 감리 문서는 "무엇이 조작됐나"(결과)는 쓰지만 "전표를 어떻게
   입력했나"(수기·승인·시각·개별금액)는 쓰지 않는다. 수단 신호가 낀 조합은 원문으로 영구 실증 불가.
   v3 발화 지도(`dev/active/phase1-rule-basis-audit/fss_rule_firing_map_v3.md`, 731행) 실측:
   구 HIGH 조합 지지 = 기말×추정계정 12~22건, 나머지 0~3건.
3. **합성데이터는 등급 근거가 될 수 없음**: "정상에서 흔하다/드물다"는 우리가 생성기에 넣은 파라미터의
   메아리(자문자답). 합성데이터의 역할은 구현 검증(S2)과 리허설 데모뿐.
4. **해소**: 시스템이 등급을 선언하지 않으면 방어할 휴리스틱이 0이 된다. 조합 선택 = 감사인의 판단
   (감사기준서 240의 분개 테스트 항목 선정도 감사인 판단 사항). 시스템 책임 = 근거 있는 어휘 제공 + 정확한 집계.

## 3. 결합 의미론

- 단위: **전표(document)** — [UNIT_MEASUREMENT_POLICY.md](UNIT_MEASUREMENT_POLICY.md) 유지.
- 기본 모드: **그룹 내 OR, 그룹 간 AND** — 선택한 몸통 중 1개 이상 AND 선택한 특징 중 1개 이상 발화한 전표.
- 엄격 모드(토글): 선택한 룰 **전부** 발화한 전표.
- 특수 케이스: 특징 0개 선택 = 몸통 조건만 적용(몸통 탐색 모드). 몸통 0개 선택 = 결과 없음이 아니라
  특징 조건만 적용하되 "몸통 미선택" 라벨 표시(모양 탐색 모드). 전부 0개 = 안내 문구 + 프리셋 제안.
- 결과 정렬: 기존 sort_key 체계 재사용(독립 신호 수 → time_severity → 금액). 등급 아님, 순서일 뿐.
