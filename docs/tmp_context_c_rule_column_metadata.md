# Context C Rule Column Metadata

기준 문서:

- `docs/tmp_context_a_rule_metadata.md`
- `docs/DETECTION_RULES.md`

목표: 룰 위반 상세 화면에서 어떤 원장 컬럼과 파생 증거 컬럼을 보여줄지 정의한다.

## Policy

- `status=active`, `presenter_surface=transaction_detail`인 canonical 룰만 row detail에서 직접 전표 위반처럼 표시한다.
- `presenter_surface=context_badge`인 active 룰은 단독 위반 상세가 아니라 다른 케이스를 보강하는 context/badge로 표시한다.
- `status=macro`, `presenter_surface=account_process_macro`인 룰은 전표 row detail이 아니라 계정/기간/모집단 단위 상세로 표시한다.
- `status=sidecar`인 룰은 관계사/graph sidecar 상세로 표시한다.
- `status=internal_reason_code`인 `L2-03a~d`는 `L2-03` drilldown 내부 reason으로만 표시하고 별도 사용자 룰 상세를 만들지 않는다.
- 원천 컬럼이 없으면 빈 값만 보여주지 말고 `fallback_message`를 표시한다.

## Column Availability Notes

- 현재 `schema.yaml`에는 `vendor_name`, `customer_name`, `counterparty`, `reference_number`, `approval_limit`, `approved_at`, `amount`, `line_amount`, `difference_value`, `expected_value`, `actual_value`, `anomaly_score`, `risk_level`, `case_id`, `priority_band`가 원천 컬럼으로 없다.
- `reference_number`는 사용자 표시명으로만 쓰고 구현 컬럼은 `reference`를 사용한다.
- 거래처 표시는 `counterparty`를 파생해 사용한다. 권장 fallback 순서는 `auxiliary_account_number`, `auxiliary_account_label`, `trading_partner`, `reference`, `"UNKNOWN_COUNTERPARTY"`다.
- 금액 표시는 `amount`를 파생해 사용한다. 권장 fallback은 `abs(debit_amount - credit_amount)`이고, 전표 합계 검산에는 `document_id` 그룹의 `debit_amount`, `credit_amount` 합계를 사용한다.
- macro finding은 `row-level anomaly_score`가 아니라 `macro_priority_score`, `review_score`, `queue_bucket`을 우선 표시한다.
- sidecar finding은 `document_id` 단일 키가 아니라 `target_document_id`와 `counterpart_document_id` 또는 graph path를 함께 표시한다.
