# 검증 결과 종합 (2026-07) — 무엇을 검증했고, 무엇이 실패했나

> 이 문서는 감사 전표 테스트 자동화 도구(Local AI Audit Assistant)의 2026-07 검증 라운드
> (S1~S5) 결과를 종합한 포트폴리오 본문이다. 성공한 검증과 실패한 검증, 그리고 도구가
> 원리상 할 수 없는 것을 같은 비중으로 기록한다. 모든 수치는 저장소 내 판정 스크립트와
> 결과 파일(§7)에서 재현 가능하며, 합성데이터 기반 수치는 전부 **개발 검증 보조 지표**다
> — 실데이터 운영 성능 주장이 아니다([CONSTRAINTS.md](../spec/CONSTRAINTS.md) 포트폴리오 주장 범위).

## 1. 한 눈에 보는 결과

도구는 세 개의 독립 검토 표면(3-surface)으로 구성되며, 세 표면은 점수를 병합하지 않는다.

| 표면                       | 역할                                          | 검증 결과                                                             |
| -------------------------- | --------------------------------------------- | --------------------------------------------------------------------- |
| PHASE1-1 룰 (31종)         | 전표 단위 규칙 위반·징후를 명명해 올림        | **성공** — 단위시험 29/29, 빌더 판정 exit 0, 부정 표면화 59.7~61.8%   |
| PHASE1-2 분석적 검토 신호  | 계정·거래처를 모아야 보이는 모집단 단위 신호  | 이번 라운드 측정 범위 밖 (별도 벤치마크 단위로 분리)                  |
| PHASE2 VAE (비지도 딥러닝) | 학습된 정상 분포 밖 비정형을 추가 후보로 제시 | **실패로 기록** — AUROC 0.5187(무작위 수준), 합성 경로 검증 자체 불가 |

여기에 더해, 검증 과정에서 도구의 **원리적 한계** 두 가지를 확정했다(§5): 순환거래
원환 탐지는 단일 법인 원장으로 불가능하고, 유령급여는 전표 룰이 아니라 급여 마스터
대사의 영역이다.

- VAE(Variational Autoencoder, 변분 오토인코더): 정상 데이터만으로 "정상의 모양"을
  학습한 뒤, 복원이 잘 안 되는 행을 이상 후보로 올리는 비지도 모델.
- AUROC(ROC 곡선 아래 면적): 순위 매김 성능 지표. 1.0이 완벽, 0.5가 동전 던지기.

## 2. 검증 방법론 — 정확도 자랑이 아니라 검증 가능한 주장

이 라운드의 목표는 높은 수치가 아니라 **방어 가능한 주장**이었다. 세 가지 원칙을
측정 전에 고정했다.

**첫째, 합성데이터의 자문자답 문제를 인정한다.** 부정 전표를 생성기로 심고 그것을
탐지기로 찾으면, 자기가 심은 흔적을 자기가 찾는 순환 논증이 된다. 따라서 합성데이터의
역할을 두 가지로 제한했다: 구현 검증(룰이 기술적으로 발화하는가)과 리허설 데모.
"정상에서 이 조합이 드물더라"는 관찰도 생성 파라미터의 메아리이므로 위험 등급의
근거로 쓰지 않았다 — 이 원칙이 §3의 등급 자동판정 폐지 결정으로 이어졌다.
부정 주입 데이터에는 지름길 차단 게이트 16종(부정 전용 계정·전용 속성값 등 라벨을
간접 유출하는 인공물 검사)을 걸어, 탐지가 구조 신호로만 가능하도록 강제했다.

**둘째, 완료 선언은 문장이 아니라 스크립트 종료 코드다(census 원칙).** "전 룰 확인
완료" 같은 주장은 판정 스크립트가 분모를 파일로 고정하고 차집합 0을 확인해 exit 0을
반환할 때만 성립한다. 판정 기준(발화율 대역, 합격선)은 측정 전에 선언해 사후 조정을
차단했다.

**셋째, 단위시험과 성능 측정을 분리한다.** 발화하라고 만든 데이터에서 발화한 것은
성능 증거가 아니다. 성능 주장은 오직 S5(부정 주입 데이터 실측)에서만 하고, S2·S4는
"쏴야 할 때 쏘는가"만 증명한다.

검증 여정 전체는 다음과 같다.

```
[S0 정상 base 생성]──▶[S1 정합성]────▶[S2 룰 단위시험]──▶[S4 빌더 단위시험]──▶[S5 성능 실측]──▶[S6 본 문서]
 355,786행, 라벨 0     정상에서 조용한가   쏠 때 쏘는가        조합 집계가 정확한가   심은 부정이 잡히나
                      룰 27/28 대역 내    29/29 exit 0        exit 0               표면화 59.7~61.8%
                      VAE 플래그 1.056%                        │                    VAE AUROC 0.5187
                                          [S3 tier 폐지·조합 빌더 재설계]가 S2와 S4 사이에 개입(§3)
```

S1에서 정상 base(s10_c001, 355,786행)에 대해 룰 28종의 발화율이 사전 선언 대역 안인지
판정했다 — PASS 27 / FAIL 1(L4-03, 생성기 손익 비정합과 룰 임계 결함 복합으로 확정 후
별도 라운드로 이연). VAE는 미학습 정상에서 플래그율 1.056%로 오염 설정(1.00%) 근처
— 사전 선언 대역 [0.3배, 3배] 안이었다. 정상에서 조용함을 먼저 확인해야 이후의 발화가
의미를 갖는다.

## 3. PHASE1 룰 표면 — 검증 성공

### 3.1 룰 단위시험 (S2): 29/29

활성 표면 룰 전수가 자기 표적 전표에서 발화하는지 판정했다. 분모는 문서의 "31종"을
믿지 않고 탐지기 레지스트리에서 스크립트로 추출해 고정했다(레지스트리 35 − 설정상
비활성 확장 6 = 29). 정상 배경 약 12만 행에 룰별 교과서 위반 전표 2~5건을 주입하고,
"발화했지만 다른 전표에서"는 실패로 세는 표적 적중 기준을 적용했다.

결과: 1차 26/29 → 미발화 3건을 추적한 결과 전부 시험 장치 결함(주입 레시피가 룰
요건을 덜 충족)으로 판명·교정 → 최종 **29/29, 룰 결함 0건, exit 0**.

### 3.2 등급 자동판정 폐지 — 실증이 무너져서 내린 결정 (여정)

구 설계는 룰 조합에 HIGH/MEDIUM/LOW/CONTEXT 등급을 자동 부여했다. 이 등급의 실증
근거를 검증하기 위해 금융감독원(FSS) 감리 지적 사례를 엄격 기준으로 재태깅(v3, 731행)
한 결과, 구 HIGH 조합의 지지가 붕괴했다: 기말×추정계정 조합만 12~22건이고 나머지는
전부 0~3건. 원인은 자료의 장르 한계다 — 감리 문서는 "무엇이 조작됐나"(결과)는 쓰지만
"전표를 어떻게 입력했나"(수기·승인·시각)는 쓰지 않으므로, 수단 신호가 낀 조합은
원문으로 영구히 실증할 수 없다. 남는 근거는 조합별 전문가 특례("보강신호 1개" 등)뿐
이었고, 조합마다 다른 사연은 방어 불가능한 휴리스틱이다.

결정(2026-07-17): 등급 자동판정을 전면 폐지하고 **조합 빌더**로 대체했다. 시스템은
근거를 표기할 수 있는 어휘만 제공하고 — 몸통 10종(금감원 확정 태깅 1건 이상인 조작
대상 룰) × 특징 10종(감사기준서 240의 부정 전표 특징 문언에 매핑되는 룰) — 어떤
조합을 볼지는 감사인이 선택한다. 감사기준서 240에서도 분개 테스트 항목 선정은 감사인
판단 사항이다. 시스템이 등급을 선언하지 않으면 방어할 휴리스틱이 0이 된다.
설계 정본: [PHASE1_COMBO_BUILDER_SPEC.md](../spec/PHASE1_COMBO_BUILDER_SPEC.md).

### 3.4 부정 표면화 실측 (S5): 60% 안팎, 프리셋별 밀도 차이가 크다

정상 base는 그대로 두고, 금감원 실증 수법 카탈로그 기반 14개 scheme(FS01~14)의 부정
330문서를 얹은 overlay 데이터셋을 난수 시드만 바꿔 4벌 생성했다(부정 밀도 0.29%,
지름길 게이트 16종 전부 통과). 측정 항목은 "부정 문서가 검토 표면에 오르는 비율"이다
— 부정 적중률이 아니라 **감사인이 볼 목록에 올라오는가**이며, 진성 라벨 대비 수치는
개발 검증 보조 지표다.

| 측정                      | 결과 (시드 4벌)                                 |
| ------------------------- | ----------------------------------------------- |
| 표면 커버리지             | 197~204 / 330 문서 (59.7~61.8%, 4벌 일관)       |
| 프리셋 "역분개·은폐"      | 적중 85~103, 표면 약 10,800문서 (밀도 0.8~1.0%) |
| 프리셋 "비용자산화"       | 적중 20~25, 표면 232문서 (밀도 8.7~10.6%, 최고) |
| 프리셋 "수익인식"         | 적중 6~10, 표면 260문서 (밀도 2.3~3.8%)         |
| 프리셋 "추정·관계사 조작" | 적중 7~11, 표면 5,505문서 (밀도 0.13~0.20%)     |

표면에 오르지 않은 약 40%는 대부분 scheme의 위장 컴포넌트 — 라벨은 부정이지만 전표
단독으로는 규칙 위반이 아닌 다리 전표 — 로, 전표 룰이 안 올리는 것이 설계 의도와
일치함을 원인 분해로 확인했다. 예컨대 매출 과대계상 주입 전표의 개별 금액은
정상 분포 안이라(총량 조작을 전표로 쪼개는 현실 은폐 방식) 금액 이상치 룰이 침묵하는
것이 도메인 귀결이다. 상세 분해: [BUILDER_PERFORMANCE_SUMMARY.md](../../reports/s5_fraud_overlay/BUILDER_PERFORMANCE_SUMMARY.md) §3.

### 3.5 검증이 실제로 잡은 결함 3건

이 검증 절차의 가치는 수치보다 결함 검출에서 확인됐다. 세 건 모두 기존 시험으로는
무증상이었고, 새 판정 축이 처음 드러냈다.

1. **L3-03 스테일 topic** (S4 1차 FAIL): 이중 레지스트리 중 한쪽만 갱신돼 2026-06에
   삭제된 topic을 참조 → 관계사 룰의 발화가 표면 진입 전에 전량 탈락. 구 등급 체계에선
   보조 신호라 증상이 안 보였고, 빌더 어휘로 승격되자 표면화. 수정 후 증거 보유 unit
   0→199.
2. **L2-05 역분개 표면 유실** (S4 V0b 실측): 탐지기 발화 135문서 중 109가 표면 승격
   게이트에서 탈락. 게이트 조건(적요·작성자 유사)이 수기 은폐형일수록 탈락시키는
   역선택임을 확인하고 룰을 수정 — 유실 109→2. 탐지기와 표면 빌더가 같은 정의를 따로
   구현하면 생기는 드리프트를 발화→표면 전수 대조가 측정했다.
3. **L3-10 추정계정 목록 갭** (S5 측정 중): 계정 체계 동기화가 1개 계정만 반영해
   추정계정 룰이 부정 데이터에서 발화 0. 대손·감가상각 등 4계정 추가 후 해당 프리셋
   적중 0~2→7~11.

기록: [debugging.md](../debugging.md) 2026-07-18 항목 3건.

## 4. PHASE2 행 단위 VAE — 검증 실패의 기록

정상 50,000행 × 71피처로 VAE를 학습(라벨 미사용)하고 부정 overlay 4벌의 전 행을
채점, 문서 단위로 평가했다.

| 지표 (시드 4벌 합산)                  | 값                      |
| ------------------------------------- | ----------------------- |
| 문서 AUROC 평균                       | **0.5187** (무작위≈0.5) |
| recall@top1% 평균                     | 0.0038                  |
| PHASE1 미표면 부정의 VAE 상위 1% 회수 | 4벌 합계 2건            |

**판정: 행 단위 VAE 검증 = 실패 (2026-07-18 확정).** 수치를 완화해 서술하지 않는다.

실패의 원인 분해가 이 절의 산출물이다.

**정보론적 귀결이지 모델 결함이 아니다.** 본 overlay는 지름길 게이트 16종으로 부정
행의 개별 피처를 전부 정상과 동일하게 강제한 "완전 위장" 시뮬레이션이다. 행 단위
피처에 정상과의 구분 정보가 없으므로, 행 단위 비지도 모델이 무반응인 것은 어떤
모델이든 피할 수 없는 결과이고 시드 4벌에서 일관 재현됐다.

**합성 경로로는 검증 자체가 불가능하다 — 딜레마.** 흔적을 심으면 자기가 심은 것을
찾는 자문자답이고, 심지 않으면(본 실측) 정보가 없어 못 잡는다. 어느 쪽도 성능 증거가
되지 못한다. 실제로 구세대 데이터(v7)에서 얻었던 AUROC 0.93은 흔적을 심은 자문자답
수치였음을 이번 라운드에서 명시적으로 무효화했다. 제3자 벤치마크(Tritscher ERP-Fraud,
문서 AUROC 0.87~0.999)는 자문자답은 아니지만 홀드아웃당 부정 표본이 6문서에 불과한
시뮬레이션이라 "코드가 동작한다" 이상의 의미를 부여하지 않는다.

파생 결정 하나: VAE 입력에서 금액 컬럼 차단(deny)을 유지했다. overlay 금액 설계가
"정상 분포 내 은폐"로 확정된 이상, 금액 피처를 풀어도 부정 신호에 기여할 정보가 없고
(원시 금액이 입력에 있던 구성에서도 무반응) 고액 정상 전표 과탐 축만 추가되기 때문이다.

이 실패 기록에 따라 VAE 표면은 "학습된 정상 밖 비정형의 추가 검토 후보 제시"라는
보조 역할로만 유지하며, 부정 탐지 성능을 주장하는 표면으로 쓰지 않는다.

## 5. 원리적 한계의 확정 — 못 하는 것에 선을 긋다

검증 라운드의 산출물에는 "이 도구가 무엇을 못 하는가"의 확정이 포함된다.

**순환거래(FS05) 원환 탐지는 단일 법인 GL(총계정원장)로 불가능하다.** A→B→C→A로
도는 자금의 원환을 보려면 자금 네트워크 전체가 보이는 시점 — 은행 계좌망, 감리당국의
다사 장부 대조, 연결 실체 — 이 전제된다. 그래프 신경망(GNN) 계열 문헌의 순환 탐지
성과도 그 전제 위의 결과라 단일 법인 범위인 본 프로젝트에 적용되지 않는다. 단일
법인에서 가능한 상한은 순환의 단면 신호(관계사 거래처와의 매출·매입 양방향 대칭,
기말 집중)를 검토 신호로 올리는 것까지이고, 실물 증빙 대사는 감사인 절차의 몫이다
— 금감원 감리의 실제 적발 방식과 같다. S5 실측에서 FS05 표면화 22%는 이 한계의 수치적
표현이다.

**유령급여(FS14)는 전표 룰의 영역이 아니다.** 정상 급여 패턴을 모사하는 부정이라
전표 모양으로는 구분점이 없다(S5 표면화 33%). 인사 마스터와 급여 지급 내역의 대사,
작성자 집중도 검토가 맞는 절차다.

한계를 확정하면 도구의 주장 범위가 명확해진다: 이 도구는 부정을 확정하지 않으며,
전수 모집단에서 감사인이 먼저 볼 검토 후보를 근거와 함께 올리는 1차 스크리닝 계층이다.

## 6. 업계 실무 정합 — 룰 주력 + ML 보조 + 감사인 판단

"비지도 딥러닝이 실패했으니 도구 구조가 틀렸는가"를 확인하기 위해 빅4 회계법인의
저널 테스트 도구를 조사했다(2026-07-18).

| 도구           | 방식                                    |
| -------------- | --------------------------------------- |
| EY Helix GLAD  | 감사인 라벨 기반 지도학습               |
| KPMG Clara     | MindBridge 앙상블 — 룰 40+ 위에 ML 보조 |
| PwC Halo       | 룰 묶음 + 의심 점수                     |
| Deloitte Omnia | 기준(criteria) 기반 분석                |

네 도구 모두 룰·기준이 주력이고 ML은 보조이며, 비지도 딥러닝 단독 탐지는 실무에
없다. 본 프로젝트의 구조 — 결정론 룰이 주 표면, VAE는 병합하지 않는 보조 표면, 조합
선택과 최종 판단은 감사인 — 는 이 관행 안에 있다. 실패한 검증(§4)이 도구를 실무
표준에서 벗어나게 만든 것이 아니라, 오히려 실무가 왜 그렇게 생겼는지를 실측으로
재확인한 셈이다.
