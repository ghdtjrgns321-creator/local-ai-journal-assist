# Local AI Audit Assistant v2.0 — 프로젝트 계획서

> Latest PHASE1 role note (2026-04-28): this overview is historical. Current PHASE1 is a first-pass rule screening layer that captures rule/policy/anomaly candidates broadly, then classifies them into exception, auditor review, and high-risk candidate queues. It is not a final fraud classifier and should not be described as if DataSynth `is_fraud` / `is_anomaly` were the sole PHASE1 truth.

## Context

감사 실증절차 전표 테스트를 로컬 환경에서 자동화하는 Python 프로젝트.
MindBridge, KPMG Clara 등 상용 AI 감사 플랫폼의 핵심 로직을 오픈소스 기반으로 재현하는 포트폴리오.
PCAOB AS 2401, ISA 240 요구사항을 구현 수준에서 커버.

**현재 상태:** Phase 1a 진행 중. DataSynth v1.2.0 데이터 생성 완료, ingest/feature/validation 구현 중.

**개발 환경:** RTX 3070 Ti (VRAM 8GB), RAM 16GB, Windows (MINGW64)

### 개요서 대비 변경점
- **Qwen2.5-Coder → Qwen3-8B 1순위**: Qwen3 Ollama 지원, reasoning 향상. Qwen2.5-Coder는 Text-to-SQL 폴백
- **직접 프롬프트 → Vanna AI 2.0**: agent-based API, 자동 Plotly, 개발시간 80% 절감
- **konlpy → kiwipiepy 단독**: JVM 의존성 제거
- **reportlab → fpdf2**: 경량화
- **LangGraph → 제거**: Vanna+PandasAI로 충분, Phase 3에서 재평가

## 5. 설계 핵심 결정

1. **BaseDetector 추상 클래스** — 모든 탐지 트랙이 `detect() -> DetectionResult` 인터페이스 공유. Phase 2~3에서 트랙 추가 시 score_aggregator 수정 최소화
2. **LLM 없이 MVP 동작** — Phase 1은 LLM 호출 0. 컬럼 매핑 실패 시 수동 UI 폴백
3. **Vanna AI 2.0** — Phase 3에서 DuckDB+Ollama+ChromaDB 네이티브 연결. DDL train 후 `vn.ask()` → SQL+Plotly 자동
4. **kiwipiepy 단독** — JVM 의존성 제거, pip install 한 줄로 완결
5. **dependency-groups** — `uv sync --group core,dashboard`로 MVP 최소 설치
