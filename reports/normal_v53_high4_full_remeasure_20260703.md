## 4. ⚠️ 회귀 발견 — L2-05(역분개) 탐지기 OOM으로 미실행

v53 파이프라인에서 **L2-05 탐지기가 메모리 부족으로 실행 실패**했다(v53 발화율 미측정, v52 2.52%):

```
L2-05 실행 실패: Unable to allocate 1.93 GiB for shape (259,042,548, 1)
```

- 원인: L2-05 거울쌍 매칭이 그룹 내 카테시안(self-join)을 수행하는데, 259M행 ≈ 16,094² 규모로 폭발.
- **계정배정 안정화의 부작용**: v52는 "500행 초과 대형그룹 282개"였으나 정상 실행. v53는 대형그룹이
  26개로 줄었지만 그중 하나가 훨씬 커졌다. 계정배정 안정화가 같은 의미의 역분개(normal accrual
  reversal 2,600건 등)를 **동일 계정으로 집중**시켜, L2-05가 매칭 키로 쓰는 그룹 하나가 ~16k행으로
  비대해졌기 때문이다(header_text 최대그룹 2,790, reference 최대 994 — 실제 매칭키 그룹은 더 큼).
- 영향: (a) L2-05 발화율 v53 미측정, (b) L2-05가 HIGH-4 강신호 집합의 한 멤버라 "기말+희소쌍+역분개"
  경로 일부가 v53에서 발화 못 함(단, 그 경로는 소수). HIGH band 수치 자체는 L3-10 중심이라 L2-05
  부재의 영향은 제한적이나, **L2-05 미실행은 독립적으로 고쳐야 할 탐지기 스케일 결함**이다.
- 조치 후보: L2-05 그룹 카테시안에 그룹 크기 상한 가드(대형그룹 청크 처리 또는 매칭키 세분화). 별도
  작업으로 분리 권장.
